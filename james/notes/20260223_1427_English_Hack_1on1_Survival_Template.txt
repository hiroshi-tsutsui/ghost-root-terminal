# 【英語不要】米国PMの『1on1』は雑談するな。沈黙を金に変える『テンプレート会話術』【年収1000万の裏技】

## まえがき：「英語で雑談」ができなきゃ死ぬと思ってへんか？

まいど、Jamesです。

いきなりやけど、お前ら「1on1（ワンオンワン）」得意か？
特に、英語での1on1。

「いや〜、Jamesさん。僕、TOEICは800点あるんですけど、**スモールトーク（雑談）**が苦手で……」
「週末何してた？って聞かれても、"Sleeping" しか言えなくて盛り上がりません……」
「現地のエンジニアと仲良くなるには、やっぱアメフトとか覚えなアカンのでしょうか？」

はっきり言うわ。
**アホか。**
その考え方してる時点で、お前は一生「英語ができない日本人」のまま終わるで。

俺は今、米国のAI企業でPMとして働いてる。年収は1000万超えてる。
でもな、俺の英語力は中1レベルや。ネイティブの早口ジョークなんか1ミリも分からんし、週末の話なんか「子供と公園行った」くらいしか言えん。

それでも、チームのエンジニアやデザイナーからは**「Jamesとの1on1は最高だ」**って言われてる。
なんでか分かるか？

俺が**「雑談」を一切しないから**や。

日本人は真面目すぎるんよ。「仲良くならなきゃ仕事が円滑に進まない」っていう謎の強迫観念がある。
でもな、ここはアメリカや。プロフェッショナルな現場や。
彼らが求めてるのは「面白い話ができる友達」ちゃうで。
**「自分の仕事を前に進めてくれる有能なパートナー」** や。

今回は、英語力ゼロの俺が、ネイティブの猛者たちと対等に渡り合い、信頼を勝ち取っている**「沈黙を金に変える1on1テンプレート術」**を全部教える。
これさえあれば、明日からの1on1、もう「天気の話」で気まずくなることはないで。

## 1. 【失敗談】「スーパーボウル」の話を振って自爆した日

偉そうに語ってるけど、俺も最初からできてたわけちゃう。
渡米直後の俺は、完全に「勘違い野郎」やった。

「よし、まずは現地の文化に溶け込まないと！」
そう意気込んで、日曜日に必死でスーパーボウルのハイライトを見たもんや。ルールも分からんのに。
で、月曜日の朝、意気揚々とチームのリードエンジニア（Mikeっていう強面のおっさん）との1on1に臨んだ。

**俺:** "Hey Mike! Did you see the Super Bowl yesterday? That touchdown was amazing, right?"
（ようマイク！昨日のスーパーボウル見たか？あのタッチダウン凄かったよな！）

**Mike:** "Actually James, I don't watch football. I was coding my side project all weekend. So, about the API specs..."
（実はジェームズ、俺フットボール見ないんだ。週末はずっと個人開発してたから。で、API仕様の話なんだが…）

**俺:** "Oh... I see..."
（あ……そっか……）

この時の気まずさ、想像できるか？
冷や汗が止まらんかったわ。
Mikeの目は完全に**「こいつ、仕事の話もろくにできんのに、なんで媚び売ってきてんねん？」**って言ってた。

その瞬間、俺は悟ったんや。
**「あ、俺ら外国人が無理して現地のカルチャーに合わせようとしても、所詮は猿真似や」** って。
そんな薄っぺらい雑談で信頼なんか得られるわけがない。
むしろ、「時間の無駄」と思われてマイナスなんや。

だから、その日を境に俺は決めた。
**「雑談は一切しない。1秒で仕事の話に入るマシーンになる」** と。

## 2. 雑談は「弱者」の戦略や。プロは「機能」で語れ。

まず、大前提を覆すで。
**「英語ができないなら、雑談なんか捨てろ。」**

考えてみ？
お前が拙い英語で「あー、イエスタデイ、アイ・ウェント・トゥ・ショッピング……」とか必死に話してる時間。
相手のネイティブからしたら**「こいつ何言いたいんやろ？ 時間の無駄やな」**って思われてるだけやで。
残酷やけど、これが現実や。

言葉の壁がある俺らが、ネイティブと同じ土俵（雑談力）で戦って勝てるわけないやん。
そこは**完全な負け戦**や。

じゃあどうするか？
**「機能（Function）」**で戦うんや。

PMとしての機能って何や？
エンジニアのブロッカー（障害物）を取り除くこと。
デザイナーの迷いを断ち切ること。
チーム全体のゴールを明確にすること。

これだけや。これ以外はノイズや。
だから、俺の1on1は、挨拶もそこそこに、いきなり本題から入る。
**「How can I help you?（俺に何ができる？）」**
この一言だけでええんよ。

---

## 3. 英語弱者のための「3つの魔法の質問」

俺が実際に使ってる、1on1のテンプレートを公開するわ。
これ、全部NotionかGoogle Docに書いて、画面共有しながら進めるんや。
（※画面共有する理由は、リスニングミスを防ぐためと、視覚情報で主導権を握るためや。これについては前回の記事も読んでな。）

### 質問①： "What is blocking you?" （何がお前の邪魔をしてる？）

これ最強や。
「最近どう？」なんて聞くな。「Fine」って返されて終わるだけや。
「ブロッカーはあるか？」と聞け。
これを聞くと、エンジニアは水を得た魚のように喋り出す。
「APIの仕様が決まってない」「デザインがまだ来ない」「AWSの権限がない」
そしたらお前は、**"Got it. I'll fix it."（分かった、俺がやる）** とだけ言えばええ。
これで信頼爆上がりや。

### 質問②： "What is top of mind for you this week?" （今週、何が一番気がかり？）

これは優先順位の確認や。
PMとして一番怖いのは、エンジニアが「どうでもいいタスク」に熱中して、リリースに間に合わんことや。
これを聞くことで、相手の頭の中を覗ける。
もしズレてたら、**"Let's prioritize X instead."（Xを優先しよう）** と修正するだけや。
難しい文法なんか要らん。「これ大事、これ後回し」って指差すだけでええ。

### 質問③： "How can I support your career growth?" （お前のキャリア、どうやって支援できる？）

これな、日本人はあまり聞かんけど、米国テック企業では必須や。
でも、毎回聞く必要はない。月に1回くらいでええ。
これを挟むだけで、**「こいつはただのタスク管理屋じゃなくて、俺の人生を考えてくれてるリーダーなんや」** っていう錯覚……いや、信頼を植え付けられる。
答えが聞き取れんかったら？
**"Please write it down here."（ここに書いてくれ）** ってドキュメント指差せばええねん。

## 4. 「ドキュメント」こそが、最強の防具であり武器や。

英語ができない俺らが、口だけで1on1乗り切ろうなんて、竹槍でB29落とそうとするようなもんや。
絶対に**「共通のドキュメント」**を用意しろ。

俺はNotionで「1on1 Dashboard」を作ってる。
そこに、さっきの3つの質問と、前回のAction Item（やることリスト）が書いてある。
1on1が始まったら、無言でその画面をシェアする。
すると、会話の主導権は**「英語が上手い奴」から「画面を持ってる奴（俺）」**に移るんや。

相手が何か言ったら、その場で俺がドキュメントに打ち込む。
もし間違ってたら、相手が「No, not X, it's Y」って訂正してくれる。
これで認識のズレもゼロになるし、議事録も勝手に完成する。

**「英語力」を「ドキュメント力」で補完する。**
これが、年収1000万PMのリアルな戦い方や。

### 具体的なNotionテンプレート例

これ、そのままコピペして使ってええで。

```markdown
# 1on1 with [Name]

## 📅 [Date]

### 🔥 Top of Mind (今週の優先事項)
- [ ] Task A
- [ ] Task B

### 🚧 Blockers (困ってること)
- [ ] Waiting for design review...
- [ ] Need access to...

### 💬 Discussion (相談事項)
- Topic 1: API Design
  - Notes: ...
- Topic 2: ...

### ✅ Action Items (ネクストアクション)
- [ ] James: Check with Design team by EOD.
- [ ] [Name]: Update the PR.
```

これを用意しておくだけで、英語が聞き取れなくても「ここ埋めていこうぜ」って指差すだけで会議が進む。
魔法やで、ほんまに。

## 5. 米国エンジニアの心理をハックせよ

なんで彼らが雑談より「機能的な会話」を好むか、考えたことあるか？
それは彼らが冷たいからちゃう。
**「コンテキストスイッチ（脳の切り替え）」を嫌うから**や。

エンジニアは、深い思考の中に生きてる。
コードのロジックを組んでる最中に、「ねえねえ、週末どうだった？」なんて話しかけられたら、積み上げた思考の塔が一瞬で崩壊するんや。
だから、1on1でもダラダラ喋られるとイライラする。

逆に、**「要件だけスパッと決めて、ブロッカーを取り除いて、5分で終わらせてくれるPM」** は神様や。
俺はあえて1on1を「15分」で設定することもある。
「君の時間は貴重だから、要点だけ話そう」って言ってな。

これ言うと、みんな目の色変えて感謝してくるで。
「Jamesは分かってる奴だ！」ってな。
英語がペラペラで30分雑談するPMより、片言でも15分で問題を解決して去っていくPMの方が、圧倒的に市場価値が高いんや。

## 結論：お前の「誠実さ」は、流暢な英語よりも価値がある。

最後に一つだけ言わせてくれ。

俺ら日本人は、英語は下手かもしれん。
でもな、**「約束を守る」「細部にこだわる」「相手を思いやる」** という点においては、世界でもトップクラスやと俺は思ってる。

1on1で大事なんは、面白い冗談を言うことちゃう。
**「あいつに言えば、必ず問題が解決する」** という安心感を与えることや。

その安心感さえあれば、文法がちゃらんぽらんでも、発音がカタカナでも、誰も文句言わん。
むしろ、「Jamesはクールだ。余計なことは喋らんけど、仕事は完璧だ」って評価されるんや。

だから、今日から「雑談の練習」なんかやめろ。
その代わりに、**「相手を助けるための質問」**を3つだけ覚えて、ドキュメント作って1on1に挑め。

それが、お前が米国テック企業で生き残り、年収1000万を掴み取るための、一番確実な「近道」やで。

ほな、また。
