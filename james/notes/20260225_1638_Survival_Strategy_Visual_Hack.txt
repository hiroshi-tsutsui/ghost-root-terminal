# 【PMの生存戦略】「英語力」より「図解力」で殴れ。DeepLとCleanShot Xで米国の議論を支配する泥臭い戦い方

まいど、Jamesです。

今回は「英語ができへんのに、なんで米国のテック企業でPMやれてんの？」という、もはや耳にタコができるくらい聞かれる質問に、本音で答えていきます。

結論から言うで。
**「英語力」なんか後回しでええ。「図解力」と「ドキュメント力」で殴るんや。**

これな、別に「英語を勉強するな」言うてるわけちゃうで。
ただな、TOEIC 900点とっても、現場のネイティブの議論についていけるか言うたら、また別の話やねん。
向こうは早口やし、スラング飛び交うし、文化的な背景知らんかったら何言うてるかサッパリわからん。
そんな中で、我々ノンネイティブがどうやって「こいつ、デキるな」と思わせるか。
その生存戦略を、今日は全部バラします。

## 1. 「流暢な英語」は幻想や。諦めろ。

まずな、この呪縛から解き放たれなアカン。
「完璧な文法で、流暢に喋らなアカン」
これ、日本人の真面目さが裏目に出てる典型例や。

現場で求められてるんは「美しい英語」ちゃうねん。「正しい意思決定」や。
プロダクトを前に進めること、チームの認識を合わせること。これだけ。
そのために必要な英語なんて、中学生レベルで十分や。

俺が最初に渡米した時なんか、ミーティングで発言するタイミングすら掴めんかった。
「Uh...」とか言うてる間に話題が変わってる。地獄やで。
でもな、ある時気づいたんや。
**「言葉で勝てへんのなら、視覚情報で勝てばええやん」** って。

## 2. 議論を支配する「CleanShot X」という名の聖剣

ここで登場するのが、俺の相棒 **「CleanShot X」** や。
これ、ただのスクショアプリやと思てたら大間違いやで。
**「認識のズレを0秒で殺す」ための最強の武器** や。

例えばな、UIの修正指示を出す時。
テキストで「ヘッダーの右端のマージンをもうちょっと詰めて、色はこっちの青にして…」って英語で書くの、ダルいやろ？
読んでるエンジニアも「Which blue?」ってなるわな。

そこでCleanShot Xや。
1. 画面をパシャっと撮る。
2. その場で矢印引っ張って「Here! 10px padding!」って書く。
3. 色コードもその場でピッカーで吸い取って貼り付ける。
4. **「Fix this like image above.」** で送信。

これだけ。所要時間10秒。
テキストなんか「Fix this」だけでええねん。
画像見たら一発でわかるから。

これでな、**「Jamesの指示はわかりやすい」** ってなるんや。
英語力ちゃうで。**「情報の解像度」** が高いねん。
ネイティブがベラベラ喋って「あの件どうする〜？」ってフワッとしてる間に、
俺がこのスクショ1枚Slackに投げたら、議論終了や。
**「Yes, this is it. Let's go.」**
これこそが、PMとしての「バリュー」やねん。

## 3. DeepLは「翻訳ツール」ちゃう。「思考の整理ツール」や。

次はDeepLな。これも使い方が重要や。
みんな「英語のメール書く時」に使ってると思うけど、それだけじゃもったいない。
俺は **「自分の思考の論理破綻をチェックする」** ために使ってる。

日本語でバーッとドキュメント書くやろ？
それをDeepLに突っ込む。
そこで出力された英語が、なんか支離滅裂やったら、それは **「元の日本語が論理的じゃない」** 証拠や。
主語が抜けてたり、接続詞がおかしかったりするんやな。

DeepLが綺麗な英語を返してくるまで、日本語の方を修正する。
**「日本語をリファクタリングする」** 感覚や。
これを繰り返すとな、英語力以前に **「ロジカルシンキング」** が鍛えられる。
結果、英語でも日本語でも、説得力のある文章が書けるようになるんや。

「DeepLに頼るなんてカッコ悪い」？
アホか。使えるもんは親でも使え。
AI企業で働いてるのにAI使わんとか、縛りプレイもええとこやで。
**「テクノロジーでハンデを埋める」**
これがテック企業のPMの流儀や。

## 4. 非同期コミュニケーションこそが、我々の主戦場

会議でネイティブと口喧嘩して勝つのは、正直キツい。
せやから、戦場を **「同期（会議）」から「非同期（ドキュメント・Slack）」に移す** んや。

会議の前には、必ずアジェンダと自分の意見をNotionにまとめておく。
図解も入れてな。
会議が始まったら、そのURLを貼る。
**「Read this first. (まずこれ読んで)」**

みんながそれを読んでる間、沈黙が流れる。
この時間が、俺のターンや。
読み終わった頃には、みんなの頭の中に俺のロジックがインストールされてる。
あとは「Any questions?」って聞くだけ。
議論のベースラインを自分で作ってしまえば、英語のハンデなんて関係ない。

会議の後も重要や。
議論が発散して「結局何決まったん？」ってなること、よくあるやろ？
すかさず議事録をNotionにまとめて、Slackに投げる。
ここでも図解を入れる。「決定事項」「ネクストアクション」を明確にする。
**「会議をファシリテートできなくても、クロージングはできる」**
最後を締めた奴が、その会議の勝者や。

ええか、若手エンジニア、そして海外を目指す同志たちよ。
英語コンプレックスで縮こまってる暇があったら、ツールを磨け。
CleanShot Xを買え。DeepL Proを契約しろ。Notionを使い倒せ。

俺もまだまだ戦ってる最中や。一緒に勝ち上がろうや。
