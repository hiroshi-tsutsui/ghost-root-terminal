# 【就活全否定】日本の「リクナビ」捨てて、GitHubと英語ドキュメントだけで米国AI企業に殴り込みかけた話（実録）

## はじめに：お前ら、まだ「志望動機」とか考えてんの？

正直に言うわ。
**「御社のビジョンに共感しました」とか、死ぬほどどうでもええねん。**

いやマジで。向こう（USテック企業の現場）からしたら、「で？ お前は何ができるん？ 幾ら儲けさせてくれるん？」これだけ。これに即答できへんかったら、どんだけ綺麗なエントリーシート書いてもゴミ箱行きや。

今日はな、日本の就活システムに絶望してる学生、あるいは「エンジニアとして飼い殺しにされそう」って震えてる若手に、**俺が実際にやった「裏ルート」** を全部バラす。

これ読んだら、もうリクナビ開けんくなるかもしれんけど、その代わり**「年収1000万」へのショートカット**が見えるはずや。

覚悟決めて読んでな。

## 1. そもそも「就活」というゲームの設定が間違ってる

日本の就活って、「自分を商品棚に並べて、誰かが手に取ってくれるのを待つ」ゲームやん？
**これ、完全に負け戦やで。**

なんでかって？
棚に並んでる時点で、お前は「その他大勢」のコモディティやから。
価格競争（初任給20万〜）に巻き込まれて終わり。

USの、特にAI企業のPM（プロダクトマネージャー）採用は違う。
**「このバグ直せる奴おらんか？」「この機能、誰か作れんか？」** っていう課題ベースで人が動く。

つまりな、**「就活（Job Hunting）」じゃなくて「商談（Business Deal）」** をしにいかなあかんねん。

### 俺がやったこと：HR（人事）を無視する

俺は就活中、人事と話したことほぼない。
なんでかって？ 人事は技術もビジネスも分からんから。「コミュ力」とか「協調性」とかいう謎パラメータでフィルターかけてくるやろ？ あんなん相手にしてたら日が暮れる。

俺が狙ったのは、**現場のエンジニア** と **PM** だけ。

具体的に何したか？
1. 志望企業のプロダクトを使う（触り倒す）。
2. バグを見つける、もしくは「こうしたらもっと良くなる」という改善案をGitHubのIssueに書く（英語で）。
3. そのリンクを、LinkedInでその会社のエンジニアに直接送りつける。

**「お前のところのプロダクト、めっちゃ好きやけど、ここイケてないから直しといたで（PR送っといたで）。どう？」**

これだけ。
これやった瞬間、相手にとってお前は「就活生」じゃなくて**「有能なコントリビューター（仲間）」** に変わる。
面接？ そんなん「いつから来れる？」の確認作業でしかないわ。

## 2. 英語は「中1レベル」でいい。ただし「ドキュメント」は神になれ

「でもJamesさん、英語ペラペラなんでしょ？」
**んなわけあるかい。**

俺のTOEIC？ 975あるけど、喋りは未だに中学生レベルやで。「This is a pen」に毛が生えた程度。
発音？ 完全にジャパニーズ・イングリッシュや。

でもな、USで働けてる。なんでか分かる？
**「非同期コミュニケーション（Asynchronous Communication）」を極めてるから** や。

### 喋るな、書け。

会議でペラペラ喋って場を支配する、みたいなスティーブ・ジョブズごっこは諦めろ。俺らには無理や。
その代わり、**圧倒的に分かりやすいドキュメント（仕様書、PRD）** を会議の前に叩きつけるんや。

- **Notion** で構造化する。
- **Mermaid** でフロー図を描く。
- **Loom** で画面操作を録画して貼る。

これやるとどうなるか？
会議が始まった瞬間、全員が俺のドキュメントを読んでる。「ここどういう意味？」って聞かれたら、「そこに書いてある通り（As described above）」って指差すだけ。

**英語力が低いなら、解像度を高めろ。**
テキストと図解は、世界共通言語や。
口で勝てないなら、指（キーボード）で勝つんや。これがエンジニア出身PMの生存戦略や。

## 3. 実践編：具体的にどう書けば「採用」されるIssueになるか？

ここで終わったらただの精神論やな。
俺が実際にUS企業に送りつけて、そのままオファー勝ち取った時の**「Issueの型」** を公開する。
これテンプレにしてええで。

### 悪い例（ゴミIssue）
> **Title:** Fix bug in login page
> **Body:** Login doesn't work sometimes. Please check.

これ送ったら即Blockされるわ。「再現手順（Reproduction Steps）は？ 環境は？ お前のPCのスペックは？」ってイライラされるだけ。
これじゃ**「お客さん（Customer）」** やねん。

### 良い例（採用されるIssue）
> **Title:** [Proposal] Enhance Login UX: Reduce Latency by 40% via Optimistic UI Update
> **Body:**
> **Context (Why):** current login process takes ~2.5s on average (measured on 3G network). This causes 15% drop-off rate based on similar industry metrics.
> **Proposal (What):** Implement Optimistic UI. Show "Success" state immediately while processing in background.
> **Implementation Plan (How):**
> 1. Modify `auth_controller.ts` to async return.
> 2. Add loading skeleton in `LoginView.tsx`.
> **Proof of Concept (POC):**
> I've created a draft PR here: [Link to PR]
> **Video Demo:**
> [CleanShot X Video URL]

これ見たPMはどう思うか？
「こいつ...ビジネスインパクト（離脱率）まで考えて、実装プランまで立てて、しかもコードまで書いてきよった...」
**即採用** や。

この差、分かるか？
前者は「バグ直して」っていうクレーム。
後者は**「俺がバグ直して売上上げたるわ」** っていう提案。

US企業が欲しいのは、後者だけや。

## 4. 武器（ツール）には金をかけろ。「無料」は甘え

ここで重要なのがツール選びや。
「WinShot使ってます」「Wordで仕様書書いてます」...
**帰れ。**

USの現場で戦うなら、向こうと同じ武器を使わな話にならん。
俺が絶対に入れてる「三種の神器」教えとく。これへの投資は、年収1000万への必要経費や。

### ① Linear（タスク管理の神）
Jiraとかいう化石を使ってる場合ちゃう。
USのイケてるスタートアップは全部Linearや。
ショートカットキーだけで全部操作できる。マウス触ってる暇あったらIssue切れ。
この「Linearのショートカットを呼吸するように使える」だけで、**「あ、こいつ分かってるな」** 感が出る。

### ② CleanShot X（スクショ界の最強）
Mac標準のスクショで満足してる奴、人生損してるで。
CleanShot Xなら、撮った瞬間に矢印入れたり、モザイクかけたり、GIF動画にしたりできる。
さっきのIssueの例で「Video Demo」って書いたけど、これをサクッと作れるかどうかが勝負の分かれ目や。
**「百聞は一見に如かず」** を地で行くツール。これないと俺は仕事できん。

### ③ Notion（脳の拡張）
ドキュメントは全部ここ。
Wordファイル送ってくる奴とは仕事せん。
Notionでデータベース作って、タスクとドキュメントを紐付ける。
**「ドキュメント駆動開発（Document Driven Development）」** の要や。

## 5. 「技術力」だけじゃ食えん。「技術 × ビジネス」の掛け算を作れ

エンジニアのみんな、耳痛いこと言うで。
**「コード書ける」だけじゃ、もう価値ないねん。**

AI（CopilotとかChatGPT）がコード書く時代やぞ？
「綺麗なコード書けます」なんて、電卓がある時代に「そろばん速いです」って自慢してるようなもんや。

じゃあ何が必要か？
**「何を（What）」** と **「なぜ（Why）」** を語れる力や。

俺がエンジニアからPMに転身した時、一番苦労したのはここ。「How（どう実装するか）」ばっかり考えてた。
でも、USのPMは違う。「この機能入れたら、**ユーザーの離脱率が何%減って、売上がいくら伸びるか**」しか見てへん。

### おすすめのポジション取り

今から狙うなら、**「Technical PM（テクニカルPM）」** 一択や。

- エンジニアの気持ちが分かる（技術的負債の怖さを知ってる）。
- でも、ビジネスの数字（KPI）にコミットできる。
- そして、英語のドキュメントが書ける。

この3つ揃ってる日本人、マジでおらん。
**ブルーオーシャンすぎて笑いが止まらんレベル** やで。
