まいど、Jamesです。

いきなりやけど、**今すぐリクナビとマイナビのアプリを消せ。**
あんなもん、日本の「兵隊アリ」を作るための洗脳装置やぞ。

「いやいやJamesさん、就活言うたらまずはリクナビでエントリーやろ？」
「自己分析して、ガクチカ書いて、OB訪問して…」

**アホか。**
そんなことしてる間に、世界のトップ層は GitHub にコード上げて、LinkedIn でスカウト待ってるんや。

俺が米国テック企業のPMとして採用に関わるとき、**「履歴書（Resume）」なんか最初の5秒しか見いひん。**
学歴？ GPA？ どうでもええ。
見るのはただ一つ。

**「お前、何作ったん？」**
これだけや。

今日は、日本の偏差値教育に毒されたお前らに、**「学歴も英語力も関係なく、米国AI企業に年収1000万で潜り込む」** ための、最も泥臭くて再現性のある "裏ルート" を教えたる。

ええか、これは「優秀な学生の話」ちゃう。
**「持たざる者」が逆転するための生存戦略** や。

## 2. 米国AI企業の採用現場：履歴書は「ゴミ箱」行き

まず現実を見ろ。
米国テック企業の求人には、世界中から何千通ものResume（履歴書）が届く。
Google や Meta なんて、倍率数百倍の世界や。

採用担当（Hiring Manager）は、1人の履歴書を見るのに何秒かけると思う？
**平均6秒** やと言われてる。
俺の感覚やと、もっと短い。**3秒** やな。

「University of Tokyo（東大）」？ ふーん。
「TOEIC 900」？ だから何？

そんなもん、画面スクロールして終わりや。
文字だけの履歴書なんて、誰も真剣に読まへん。

ほな、何で判断してるか。
それは **「ポートフォリオ（成果物）」** へのリンクがあるかどうか、や。

- **GitHub の URL**
- **自分で作った Web アプリの URL**
- **Zenn や Qiita の技術記事の URL**

これがあるだけで、お前は「その他大勢の有象無象」から、「中身のある人間」に昇格する。
クリックして、動くモノがあって、コードが綺麗なら、**その瞬間に面接確定** や。

日本の就活生が必死こいて「サークル副代表で〜」とか「バイトリーダーで〜」とか作文してる間に、
裏ルートを行く奴は **「URL 1本」** で勝負を決めてるんや。

## 3. 「Github履歴書」の作り方：草を生やすな、ドキュメントを書け

「わかった！ GitHub に草（コミットログ）生やしまくればええんやな！」

…ちゃうねん。それも間違いや。
毎日適当なコミットして緑色に塗りつぶしても、中身がスカスカなら意味ない。

米国PMが見てるのは、**「こいつはチームで働けるか？」** という点や。
独りよがりなスパゲッティコードなんか、マイナス評価にしかならん。

### ① README.md が本体や
リポジトリのトップにある `README.md`。これが実質的なお前の履歴書や。
ここに何を書くか？

- **What (何を作ったか)**: 一言で。GIFアニメかスクショを貼れ。
- **Why (なぜ作ったか)**: どんな課題を解決したかったんか。
- **How (どう作ったか)**: 技術スタック（React, Next.js, Python, AWS etc.）。
- **Role (自分の役割)**: チーム開発ならどこを担当したか。

特に **「GIFアニメ」** は必須や。
動いてる様子がわからんアプリなんか、誰もクローンして動かしてみようなんて思わへん。
**CleanShot X** とかでサクッと録画して、貼っとけ。それだけで「お、こいつ気が利くな」ってなる。

### ② Issue と Pull Request で「仕事の流儀」を見せろ
個人開発でも、あえて **Issue** を立てて、**Pull Request (PR)** を作ってマージしろ。
コミットメッセージを `fix` とか `update` だけで済ますな。

- `feat: Add user authentication using Firebase`
- `fix: Resolve layout bug on mobile safari`

こういう「実務っぽい」ログが残ってると、採用担当は安心する。
「あ、こいつ入社しても即戦力で回せるな」と錯覚させられるんや。

### ③ 英語で書け（DeepLでええ）
当たり前やけど、米国企業狙うならドキュメントは全部英語や。
「英語苦手なんで…」とか言うてる場合ちゃう。
**DeepL** に突っ込んで、そのまま貼り付けろ。
コード内のコメントも英語や。
文法なんか多少間違っててもええ。「英語でアウトプットしようとする姿勢」が見えたら、それで合格点や。

## 4. 戦略的インターン：日本のベンチャーを「踏み台」にしろ

いきなり米国の本社に突撃するのは、ハードルが高いかもしれん。
せやったら、**「外資系企業の日本支社」** か **「英語公用語の日本のベンチャー」** を狙え。

ここで重要なのは、**「エンジニアインターン」** として潜り込むことや。
営業とかマーケティングじゃアカン。
**「コードが書ける」「プロダクトが作れる」** という実績を作るんや。

### 具体的なアクションプラン
1. **Wantedly** で「英語」「グローバル」「インターン」で検索しろ。
2. 応募メッセージには、さっき作った **GitHub の URL** を貼れ。
3. 面接では「将来は米国で働きたいんで、御社で修行させてください」と正直に言え。
   （見込みのあるベンチャーなら、そういう野心のある学生を面白がってくれる）

そこで半年でも実務経験積んで、プロダクトのリリースに関わったら、それが最強の武器になる。
その経験を引っ提げて、LinkedIn で **"Software Engineer Intern"** とプロフィールを書き換えろ。
すると不思議なことに、海外のリクルーターから連絡が来るようになるんや。

## 5. LinkedIn ハック：日本の就活サイトなんか見るな

リクナビを見るなと言った理由はこれや。
世界のジョブマーケットは **LinkedIn** で回ってる。

まだアカウント作ってない？ **論外や。** 今すぐ作れ。

### プロフィールの書き方
- **Headline**: "CS Student at XX Univ | Aspiring Software Engineer | React/Next.js"
  （「学生です」じゃなくて「何ができるか」を書け）
- **About**: 英語で書け。DeepLでええ。
  「I am passionate about building scalable web applications...」みたいなテンプレで十分や。
- **Featured**: ここに一番自信のある **GitHub のリポジトリ** や **ポートフォリオサイト** のリンクを貼れ。サムネイル付きでな。

### ネットワーキング
- 志望する企業のエンジニアやPMに「Connection Request」を送れ。
- メッセージなしで送るな。「あなたの記事を読みました」「あなたのキャリアに憧れてます」と一言添えろ。
- 返事が来なくても凹むな。数打ちゃ当たる。
