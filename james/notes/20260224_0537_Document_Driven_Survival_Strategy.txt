# 【完全版】凡人が「米国テック企業」で生き残るための「ドキュメント駆動」生存戦略（英語力ゼロでも勝てる唯一の武器）

## はじめに：なぜ、あなたの英語は「伝わらない」のか？

まいど、Jamesです。

今日はちょっと本気の話をします。「米国テック企業で働きたいけど、英語が怖い」とか「留学経験ないし無理」って思てる人、正直に手を挙げてください。

…うん、画面の前で結構手が挙がった気がするわ。

結論から言うで。「英語力」で勝負しようとしてる時点で、その戦いは負け確や。
俺らみたいな純ジャパが、ネイティブと「会話」で殴り合って勝てるわけがない。発音？ 流暢さ？ そんなもん、向こうの小学生以下やで、俺らは。

せやけど、俺は今こうして米国のAI企業でPMとして飯を食うてる。年収もそれなりに貰うてる。
なんでか分かるか？

**「喋り」を捨てて、「書く」に全振りしたからや。**

もっと正確に言うとな、「ドキュメント」という武器を使って、**非同期（Async）の戦場**に相手を引きずり込んだんや。
今日は、俺が実際に現場で使ってる「ドキュメント駆動（Document-Driven）」の生存戦略を、包み隠さず全部公開するわ。これを読めば、TOEICの点数なんか1ミリも変わらんくても、明日から「仕事ができる奴」に変われる。約束するで。

---

## 1. 「英語会議」は日本人の墓場やと思え

まず、大前提の話をするで。
日本人が一番やらかすミス。それは「会議でなんとか爪痕を残そうとする」ことや。

無理やて。
向こうの会議、マジで早口言葉の大会やから。
ネイティブが5人集まって、バーーッて喋りまくって、最後に「So, what do you think, James?」って急にパス投げてくる。
ここで「Uh... well...」とか言うてる間に、もう次の話題行ってるわ。

**リアルタイムの音声コミュニケーションは、俺らにとって「敵のホームグラウンド」や。**
そこで戦ったらアカン。

### 戦場を変えるんや

俺らが勝てるのはどこか？
それは**「テキスト」と「図解」の世界**や。

コードは嘘つかんやろ？ 仕様書も嘘つかん。
どれだけ早口なアメリカ人でも、ドキュメントを読むスピードは俺らとそんなに変わらん（いや、速いけど、喋りよりはマシや）。
何より、テキストなら「Google翻訳」も「DeepL」も、最近なら「ChatGPT」も使い放題や。

**「喋ったら負け。書いたら勝ち。」**
これを合言葉にしてほしい。

## 2. James流「ドキュメント駆動」3つの鉄則

ほな、具体的にどう戦うか。俺が実践してる3つの鉄則を教えるわ。

### 鉄則①：会議の前に「勝負」を決めておく（Pre-readの魔力）

会議が始まってから「今日の議題は〜」って言うてる時点で、もう遅いねん。
俺は会議の24時間前、最低でも1時間前には、**「議論の全て」を書いたドキュメント（1-pager）**をSlackに投げる。

内容はこうや。
1. **Context（背景）**: なぜこの話をするのか。
2. **Problem（課題）**: 今何が問題なのか。
3. **Options（選択肢）**: A案、B案、C案。（メリット・デメリット付き）
4. **Recommendation（推奨案）**: 俺はB案で行きたい。なぜなら〜。

これを投げて、「会議ではこれをベースに話すで。読んどいてや」って言うとくねん。
そうすると何が起きるか？

会議が始まった瞬間、みんなそのドキュメントを見てる。
俺は拙い英語で一言言うだけでいい。
**"As I wrote in the doc, I recommend Option B. Any concerns?"** （ドキュメントに書いた通り、B案推奨やけど、なんか懸念ある？）

これだけ。
あとはネイティブたちが勝手に「B案のここはリスクじゃないか？」「いや、A案よりはマシだ」って議論し始める。
俺はそれをニコニコ聞いて、最後に **"Okay, let's go with B."** って言うだけ。

これが「PMの仕事」や。喋り倒すことじゃない。意思決定をドライブすることや。

### 鉄則②：視覚情報で「英語」を殺す（CleanShot X 活用術）

ドキュメント書く時、まさか文字だけで埋めてないやろな？
英語が苦手な奴ほど、長文を書きがちや。アカンて。読んでくれへんて。

**「スクリーンショット」と「矢印」こそが、俺らの共通言語や。**

俺が愛用してる『CleanShot X』っていうMacアプリがあるんやけど、これが最強。
バグ報告とか仕様説明とか、画面キャプチャ撮って、その上に矢印引っ張って、「Click here」とか「This is wrong」って書くだけ。

- **Bad**: "When user clicks the submit button, an error occurs because the API returns 404..." (長い、読む気失せる)
- **Good**: [ボタンに赤枠] → [矢印] → [404エラーの画面] （一瞬で伝わる）

**「一目で分かる」は「流暢な英語」より100倍価値がある。**
俺のドキュメント、半分くらい画像やで。でもそれが「Jamesのドキュメントは分かりやすい」って評価されるんや。皮肉なもんやろ？ 英語減らした方が評価されるんやから。

### 鉄則③：議事録は「リアルタイム」で画面共有して書く

これ、最強のライフハックやから覚えて帰って。
英語の会議で一番怖いのが、「何が決まったか分からんまま終わる」ことやろ？
あとで「言った言わない」になるのが一番ダルい。

せやから俺は、会議中 **ずっとNotionとかGoogle Docsを画面共有しとくねん。**
で、議論を聞きながら、その場で箇条書きにしていく。

- Action Items:
    - [ ] Fix the login bug (@Mark)
    - [ ] Update the design (@Sarah)

もし聞き取れんかったら？ その場で書く手を止めて聞くねん。
"Sorry, what was the action item for Mark?"
画面共有してるから、みんな俺がタイプするのを待ってくれる。
もし俺が間違ったこと書いたら、"No, no, not login bug, it's logout bug." って誰かが突っ込んでくれる。

**「書記」をやるフリして、会議の主導権（ファシリテーション）を握るんや。**
画面共有されてるテキストが、その会議の「正義」になる。
終わった瞬間、"Thanks everyone, I'll share this note." で終わり。議事録作成の時間ゼロ。完璧やろ？

## 3. ツールへの投資をケチるな（ROIの話）

ここまで読んで、「なるほどな〜」で終わらせたらアカンで。
このスタイルを実現するには、**「道具」への投資**が不可欠や。

エンジニアとかPMって、ここケチる人多いんよな。「無料ツールでええやん」って。
あかんで。プロなら商売道具には金かけな。

### 必須ツール三種の神器

1.  **CleanShot X ($29/lifetime)**
    *   さっき言うた神スクショアプリ。これがないと俺は仕事できん。
    *   画面録画（GIF）も一瞬で作れるから、「再現手順」とか動画で送れる。
    *   「英語で説明するコスト」を考えたら、29ドルなんか1日で回収できる。

2.  **Grammarly (Pro版)**
    *   英語の添削ツールやけど、無料版じゃ足りん。Pro版は「トーン（口調）」まで直してくれる。
    *   俺の書く英語はぶっきらぼうになりがちやけど、こいつが "Confident but polite" に直してくれる。
    *   これはいわば**「清潔なスーツ」**や。中身が良くても、ヨレヨレのシャツ着てたら信用されんやろ？ 英語のミスはそれと同じや。

3.  **Notion / Obsidian**
    *   自分の「第二の脳」。
    *   会議の議事録、技術メモ、全部ここに突っ込む。
    *   俺は「英語のフレーズ集」もここに貯めてる。"Let's circle back on this" とか、便利そうな言い回しを聞いたら即メモって、次の会議で使う。パクって自分のモノにするんや。

## 結論：英語コンプレックスを「武器」に変えろ

長々と書いたけど、要はこういうことや。

**「英語ができない」ことを嘆くな。**
**「英語ができないからこそ、ドキュメントを書く」という行動に変えろ。**

ネイティブは、英語が上手すぎてドキュメントを書くのをサボる傾向がある。
「喋れば分かるやろ」って思てるからな。
でも、リモートワーク全盛の今、後から見返せない「口頭の合意」なんてゴミや。
歴史に残るのはドキュメントだけや。

せやから、皮肉なことに、**「英語が苦手で、必死にテキストに残そうとする日本人」**の方が、結果的に信頼されるPMになれるチャンスがあるんや。

俺はこれを「生存戦略」と呼んでる。
華麗なプレゼンなんていらん。
泥臭く、地味に、でも確実に「ログ」を残す。
それが、異国の地で俺らが戦う唯一の方法や。

今日から会議の前に1枚、メモを書いてみ？
世界変わるで。

ほな、また。
