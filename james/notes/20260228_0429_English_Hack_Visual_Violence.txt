## 2. 戦術①：『図解暴力（Visual Violence）』

会議で口数少ないのに、めちゃくちゃ影響力あるやつっておるやろ？
あれな、**「画面共有」の覇者**なんや。

英語が苦手なやつほど、口で説明しようとする。
「Uh... well... I think the user flow is like...」
**遅い！**
そんなもん聞いてる間に、ネイティブPMは次の議題に行ってるわ。

俺がやってるのはこれや。

### A. CleanShot X で「視覚的」に殴る
口で説明する前に、**スクリーンショットに矢印と文字を書き込んで、チャットに貼り付ける。**
これだけや。

例えば、UIの改善案を出す時。
口で「The button should be red and larger」とか言うな。
**今のUIのスクショ撮って、赤枠で囲って、矢印引っ張って「Color: #FF0000, Size: 24px」って書いた画像を貼るんや。**

これな、**「Visual Violence（視覚的暴力）」**って呼んでるんやけど、効果絶大やで。
画像ドーン！って貼られたら、全員の目がそこにいく。
ネイティブもノンネイティブも関係ない。「あ、これね」で終わる。
英語力ゼロでも、**「こいつは答えを持ってるやつだ」**と認識される。

### B. Mermaid.js で「構造」を支配する
複雑な仕様の話になった時、口頭で議論すな。迷宮入りするぞ。
俺は会議中に**Notion**か**Markdownエディタ**を開いて、リアルタイムで**Mermaid記法**でフローチャートを書く。

```mermaid
graph TD
    A[User Login] -->|Success| B[Dashboard]
    A -->|Fail| C[Error Page]
    C --> D[Retry Limit?]
```

これを画面共有しながら、「Is this logic correct?」って聞くだけ。
相手は「Yes」か「No」しか言えん。
**議論の土俵（構造）をこっちが作ってしまえば、英語がつたなくても主導権は握れるんや。**

## 3. 戦術②：『非同期戦術（Async Warfare）』

「会議で発言できない」？
せやったら**会議の外で勝負せぇ**。

米国テック企業、特にリモートワークが進んでる会社では、**「同期（会議）」より「非同期（ドキュメント）」の方が評価される**傾向がある。
なぜなら、ドキュメントは資産として残るからや。

### A. "Pre-Meeting" アタック
会議が始まる**24時間前**に、議題に対する自分の意見、データ、解決案をまとめたドキュメント（NotionとかGoogle Doc）を送りつける。
タイトルは **"[READ AHEAD] Proposal for Feature X"** とかでええ。

中身はDeepLとChatGPTで推敲しまくった、完璧な英語や。
会議が始まったら、「詳しくはドキュメントに書いたけど、要点は3つだ」って言うだけ。
これで、**「お前は英語ができないやつ」じゃなくて、「準備周到で仕事が速いやつ」になる。**

### B. "Post-Meeting" 追撃
会議中にうまく言えなかったこと、聞き取れなかったこと、あるやろ？
終わった瞬間にSlackで追撃するんや。

「さっきの件、再考したけど、リスクがあると思う。理由はこれ。」
「念の為確認やけど、次のアクションアイテムはこれで合ってる？」

これを**会議終了後5分以内**にやる。
相手からしたら、「会議中は静かだったけど、こいつめちゃくちゃ考えてるやん」ってなる。
**口の速さじゃなくて、レスの速さで勝つんや。**
