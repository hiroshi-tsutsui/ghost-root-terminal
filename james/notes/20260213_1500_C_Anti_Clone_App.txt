# 未経験エンジニアの「ポートフォリオ」が全部ゴミに見える理由

## 戦略分析メモ
- **ターゲット:** [C] エンジニア志望・駆け出しエンジニア
- **仮想敵:** 「スクールの課題で作ったクローンアプリ」「Progate周回アピール」
- **Jamesの逆張り:** 「コード書く前に『課題』探せ。技術力より『解決力』や。」

---

最近、X（Twitter）見ててホンマに思うんやけどな。

未経験からエンジニア目指してる子らのポートフォリオ、**全部同じに見えるんよ。**

「タスク管理アプリ作りました（ログイン機能付き！）」
「ECサイトのクローン作りました（カート機能付き！）」

…いや、それ誰が使うん？

厳しいこと言うけど、採用担当のPMからしたら**「あ、またこれ系か」でブラウザバック確定**やで。

スクールのカリキュラム通りに作っただけの成果物なんて、ポートフォリオちゃう。ただの「写経」や。

今日は、そんな**「量産型エンジニア」から抜け出して、米国テック企業でも通用する『本物のポートフォリオ』の作り方**を教えたるわ。

## 1. 「技術力」アピールは無意味やと思え

まず大前提として、未経験の技術力なんてたかが知れてる。

どんだけ綺麗にReact書けても、実務経験3年のエンジニアには勝てへん。そこを競おうとするから埋もれるんや。

日本の採用現場やと「学習意欲」とか見てくれるかもしれんけど、グローバル基準やと**「で、君は何ができるん？」**で終わり。

重要なのはコードの綺麗さちゃう。**「そのアプリで誰の何を解決したか」**や。

## 2. 俺が実際に見た「採用したくなる」PF

昔、俺が採用に関わった時に「おっ」と思った未経験の子がおった。

その子が作ったのは、**「近所のスーパーの半額弁当を通知するLINE Bot」**やった。

技術スタックはショボい。PythonのスクレイピングとLINE APIだけ。DBすら使ってない。

でもな、その子はこう説明したんや。

「自分は貧乏学生で、毎日スーパーに通う時間が無駄でした。だからこのBotを作って、通知が来た時だけ行くようにしたら、月30時間の節約になりました。」

これよ。これがエンジニアリングや。

**自分の課題を、技術を使ってハックする。**
この姿勢が見えるやつは、言語が変わろうがフレームワークが変わろうが、現場で絶対活躍する。

## 3. James流：AI時代の「勝ち確」ポートフォリオ戦略

ほな、具体的に何作ればええねんって話やな。

今からイチから勉強してRailsとかNext.jsやるんもええけど、もっと賢いやり方がある。

**「AI API × ニッチな課題解決」**

これ一択や。

例えば、「OpenAI APIを使って、長文の利用規約を3行で要約してくれるChrome拡張」とかどうや？

技術的にはAPI叩くだけやから簡単。でも、**「読むのが面倒」という人類共通の課題**を解決してる。

こういう「小さくても実際に使えるツール」を作って、実際にユーザー（友達でもええ）に使わせる。
そのフィードバックを受けて改善したログを残す。

ここまでやって初めて、ポートフォリオは「作品」から「実績」に変わるんや。

## 4. 今日からやること

スクールの課題が終わったら、PC閉じて街に出ろ。

- バイト先の店長が困ってることはないか？
- 親がスマホ操作でイライラしてることはないか？
- 自分自身が「めんどくさい」と思う作業はないか？

その「めんどくさい」が全部ネタになる。

**コードを書く前に、課題を見つけろ。**

技術はそのための手段に過ぎへん。
手段と目的を履き違えてるうちは、いつまで経っても「その他大勢」のままやで。

ほな、また。
