# 米国PM流：英語力「中1レベル」でも年収1000万を超える "泥臭い" 生存戦略

## まえがき：英語ペラペラ信仰を捨てろ

「英語ができないから、海外企業なんて無理です」
「TOEIC 900点取ってから挑戦します」

アホか。そんなん待ってたら、爺さんになってまうで。

正直言うとな、僕の英語力なんて **中1レベル** や。
SVO（主語・動詞・目的語）しか喋れへん。関係代名詞？ 何それ美味しいの？ ってレベル。
発音だってバリバリの日本語訛りやし、流暢な会話なんて夢のまた夢。

それでも僕は今、米国のAI企業でプロダクトマネージャー（PM）として働き、年収1000万以上を稼いでる。
現地のネイティブエンジニアたちを束ねて、ゴリゴリにプロジェクト回してるんや。

なんでこんなことが可能なんか？

答えは簡単や。**「英語力」と「仕事力」は別物** やからや。

多くの日本人はここを勘違いしてる。「英語が上手くなれば、グローバルで働ける」と思い込んでる。
逆や。**「仕事を進めるための "武器" を持っていれば、英語なんて最低限でいい」**。

今日は、僕が実際に現場で使ってる **「英語弱者のための生存戦略」** を全部晒す。
これは綺麗な英語を話すためのノウハウやない。
**「ブロークンな英語でも、相手を動かし、成果を出し、高給を勝ち取るための泥臭い戦術」** や。

もし君が「英語の勉強」に逃げて、キャリアの挑戦を先延ばしにしてるなら、この記事を読んで目を覚ましてほしい。
必要なのは語学力やない。**「伝わらなくても伝える」という執念と、それを支えるツール活用術** や。

## 1. 非同期コミュニケーションという「最強の防壁」

まず大前提として、**リアルタイムの英会話で勝負しようとするな**。
ネイティブ相手に口喧嘩で勝てるわけがない。
会議で気の利いたジョークを言う？ 無理無理。そんなもん諦めろ。

僕らの主戦場は **「テキスト」** や。
Slack、Notion、Jira、GitHub。ここなら勝てる。
なぜなら、**「考えてから発信できる」** からや。

### 即レス禁止。一呼吸置いて「DeepL」を通せ

僕が徹底してるのは、Slackでメンションが来ても **「即レスしない」** こと。
焦って変な英語を返すのが一番アカン。

1.  相手のメッセージをDeepLに放り込む（ニュアンスの確認）。
2.  日本語で論理的な返信を書く。
3.  DeepLで英訳する。
4.  **ここが重要：** 英訳された文を「逆翻訳」して、日本語として意味が通るか確認する。

これをやってから送信する。慣れればこのサイクルは10秒で回せる。
これだけで、誤解の9割は防げるんや。

### 「ミーティング」を「ドキュメントレビュー」に変える

英語弱者にとって、一番の地獄は「何が決まったかわからないまま終わる会議」や。
これを防ぐために、僕は **「ミーティングを極力開かない」** 戦略を取ってる。

議論したいことがあったら、まず **Notionでドキュメントを書く**。
背景、課題、提案、メリット・デメリット。全部書き出す。
そして、そのリンクをSlackに貼ってこう言うんや。

> **"Please read this doc and leave comments by EOD. If needed, we can have a short sync."**
> （これ読んで今日中にコメントして。必要なら短く話そう。）

これなら、相手もテキストでコメントしてくれる。
テキストならDeepLが使える。Google翻訳が使える。
**自分の土俵（テキスト）に相手を引きずり込む** んや。これがPMとしての生存戦略の基本中の基本。

## 2. 「図解」は英語の100倍伝わる

英語が下手な奴ほど、言葉で説明しようとして自爆する。
ええか、**米国のエンジニアは忙しい**。
君の拙い英語の長文なんて読んでる暇はないんや。

そこで最強の武器になるのが **「図解（Visual）」** や。

### CleanShot X を使い倒せ

Macユーザーなら **CleanShot X** は必須や。これを入れてないPMはモグリやと言ってもいい。
スクリーンショットに矢印、枠線、番号、ぼかしを入れる作業が、一瞬でできる。

例えば、UIの修正指示を出すとき。
「右上のボタンを少し左にずらして、色を赤にして…」なんて英語で書くな。
**スクショを撮って、該当箇所を赤枠で囲んで、矢印を引っ張って "Change to Red" と書く。**
これだけでいい。これ以上の説明はいらん。

### Mermaid でロジックを可視化しろ

複雑な仕様やフローを説明するときは、**Mermaid記法** を使う。
NotionやGitHubなら標準対応してるから、テキストベースでフローチャートが描ける。

```mermaid
graph TD
    A[Start] --> B{Is Valid?}
    B -- Yes --> C[Process Data]
    B -- No --> D[Return Error]
```

こういう図をポンと貼るだけで、エンジニアとの共通言語ができる。
「Is Valid? の条件は何？」とか、図を指差しながら話せるから、英語の負担が激減するんや。
**「絵」は世界共通語**。これを使いこなせるだけで、英語力のハンデはチャラにできる。

## 3. 面接・1on1 は「台本」を用意しろ

そうは言っても、どうしても話さなきゃいけない場面はある。
採用面接、上司との1on1、緊急のトラブル対応。

ここでの対策は一つ。**「台本（Script）」を完璧に用意する** ことや。

### 想定問答集（FAQ）をNotionに作る

僕は転職活動の時、想定される質問を100個リストアップした。
「自己紹介」「強み・弱み」「過去の失敗談」「なぜこの会社か」…。
それに対する回答を、**すべて英語で書き起こして、暗記した**。

本番では、その引き出しを開けるだけ。
「あ、その質問はNo.32のパターンやな」と脳内で検索して、再生ボタンを押す感覚や。
これなら、流暢に喋れる（ように見える）。

### 画面の端にカンペを置け

オンライン会議なら、カメラのすぐ近くに **カンペ（Cheat Sheet）** を置いておけ。
Notionのウィンドウを細くして、画面の端に表示させておくのも手や。
挨拶の定型文、よく使う接続詞（However, Therefore, On the other hand...）、困った時のフレーズ。

> **"Let me double check specifically, I will get back to you on Slack."**
> （詳細確認させて、Slackで戻すわ。）

この魔法の言葉さえあれば、どんな難局も一旦持ち帰れる。
**「即答できない＝無能」ではない。「適当に答える＝無能」なんや**。
堂々と持ち帰れ。そしてテキストで完璧な回答を返せ。それで評価は落ちない。

## 4. 「専門用語」と「ドメイン知識」で殴れ

最後に、これが一番重要かもしれん。
**「英語」ではなく「専門用語（Jargon）」で話せ**。

IT業界のいいところは、**用語が世界共通** なところや。
React, AWS, Kubernetes, API, Latency, Scalability, ROI, KPI...
これらの単語は、日本でもアメリカでも同じ意味で通じる。

文法なんて適当でいい。
**"Database is slow. Need cache. Redis is good?"**
これだけで通じるんや。
（SVOすら怪しいけど、エンジニアには100%伝わる）

逆に、一般的なビジネス英語（敬語や遠回しな表現）なんて使うな。
**"I was wondering if you could possibly considering to..."**
なんて言ってる間に、相手はあくびしてるわ。

**"Fix this bug ASAP. Critical for release."**
これでええんや。
**プロフェッショナルな現場ほど、コミュニケーションは簡潔（Concise）であることが好まれる**。
丁寧さよりも、**「正確さ」と「速さ」**。
そこに君の価値がある。

## 結論：英語コンプレックスを武器に変えろ

勘違いせんといてほしいのは、僕は「英語の勉強をするな」と言ってるわけやない。
長期的に見れば、英語力はあった方がいいに決まってる。
でも、**「英語ができるようになるまで挑戦しない」のは、ただの機会損失** や。

完璧な英語なんて待つな。
今の君の、その拙い英語のままで、世界に飛び込め。
その代わり、**「ツール」** を使いこなせ。**「準備」** を徹底しろ。**「専門性」** を磨け。

**「英語ができないからこそ、ドキュメントを誰よりも丁寧に書く」**
**「英語ができないからこそ、図解で一発でわかる仕様書を作る」**

その工夫こそが、結果として君を **「超優秀なPM」** に変えるんや。
ネイティブのPMが口頭で適当に済ませる仕事を、君は緻密なドキュメントで補完する。
それがチームにとってどれだけありがたいか。

**君の「弱点」は、裏返せば最強の「武器」になる。**

さあ、今日からDeepLとCleanShot Xを相棒にして、世界へ殴り込みに行こうや。
年収1000万の世界は、意外と君のすぐそばにあるで。
