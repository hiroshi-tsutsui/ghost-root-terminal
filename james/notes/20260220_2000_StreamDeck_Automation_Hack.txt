## なぜ「物理ボタン」なんか？

キーボードショートカット信者の僕が、あえて手を離してボタンを押す理由。
それは **「脳のモード切り替え」** や。

キーボードは「入力」のためのデバイス。思考を文字にする場所。
対してStream Deckは **「環境」を制御するコックピット** やと思ってる。

- 「今から会議モード」
- 「今から集中執筆モード」
- 「今から退勤（店じまい）モード」

この「モード」を切り替える時、物理的なボタンを「カチッ」と押し込む感覚が、脳に「よし、やるぞ」っていう合図を送るんよ。精神論っぽく聞こえるけど、これマジで大事。

ほな、具体的な設定（レシピ）いくで。

## Level 1: Zoom/Meetの「物理ミュート」とステータス表示

これは基本中の基本やけど、一番事故らんやつ。

会議中、画面共有してる時にミュートボタン探してマウスカーソル迷子になったことない？
あれ、PMとしては致命的にダサいねん。

### James流の設定
Stream Deckのプラグインストアにある公式の「Zoom」プラグインを使うんやけど、工夫が必要や。

1. **トグルボタンにする:**
   - 押すたびに「Mute / Unmute」が切り替わる。
   - アイコンは自分で作る。「赤背景にマイク斜線」と「緑背景にマイク」みたいに、**視界の隅に入ってるだけで状態がわかる** 色使いにするのがコツ。

2. **「退出」ボタンは赤ドクロ:**
   - 間違って押さんように、右上の端っこに配置。
   - これを押したら `Cmd+W` を送信して即離脱。挨拶してからの0.5秒のラグすら許さん。

**【効果】**
「あ、すんませんミュートでした」っていうあの無駄な3秒が人生から消滅する。

## Level 2: "Deep Work" プロトコル（AppleScript連携）

ここからが本番や。
「集中したいからSlack閉じて音楽かけよ」って手動でやってたら、その間に来た通知見てまうやろ？
**集中に入るための準備で気が散る** っていう本末転倒なことが起きる。

せやから、ボタン一発で「強制的に集中環境」を作るねん。

### 使うもの
- Stream Deck (System > Open)
- AppleScript (またはShell Script)

### レシピ: "The Bunker" (地下壕モード)
僕が「Bunker」って呼んでるボタンがある。これを押すと、以下のことが**一瞬で**起きる。

1. **Slack / Discord を強制終了 (Quit)**
   - 「通知オフ」やない。「終了」や。プロセスを殺す。アイコンが見えてたら気になってまうからな。
2. **"Do Not Disturb" (おやすみモード) ON**
   - 通知バナーなんか言語道断。
3. **Spotifyで「Lo-Fi Hip Hop」プレイリストを再生**
   - 音量は指定のレベル（20%）に固定。
4. **Obsidian（執筆用エディタ）をフルスクリーンで起動**

これを実現するAppleScriptを書いて、`.scpt` ファイルとして保存。Stream Deckの「Open」アクションでそのファイルを指定するだけ。

**【AppleScriptの例 (コピペしてええで)】**
```applescript
tell application "Slack" to quit
tell application "System Events"
    -- おやすみモードON (Shortcutsアプリ経由が確実やけど簡易版で)
    keystroke "focus_mode_shortcut" using {command down, option down} 
end tell
tell application "Spotify"
    play track "spotify:playlist:xxxxxx"
    set sound volume to 20
end tell
tell application "Obsidian"
    activate
    tell application "System Events" to keystroke "f" using {command down, control down} -- フルスクリーン
end tell
```

これを押した瞬間、世界が静かになる。**「ゾーン」への入り口を物理化する** ってことや。

## Level 3: "Review Request" 爆撃ボタン (Text Expansionの物理化)

PMやってると、エンジニアにPR（Pull Request）のレビュー依頼とか、デザインのフィードバックとか、定型文を打つ回数が尋常じゃない。

Espanso（以前紹介したやつ）でもええんやけど、Stream Deckの「Multi Action」を使うと、**複数のステップ** を自動化できる。

### シナリオ: Slackでエンジニアにレビュー依頼を投げる
1. **ボタン「Review」を押す**
2. クリップボードにあるURL（JiraやGitHubのリンク）を保持。
3. 自動でSlackの特定のチャンネル（#dev-review）を開く（URLスキーム使用）。
4. 以下のテキストを自動入力:
   > @channel お疲れ様です。以下の件、レビューお願いします！ :pray:
   > 期限: 本日17:00まで
   > [URL]
5. `Enter` で送信。

これ全部、ボタン一発。
**「チャンネル移動 → 文字入力 → 送信」** という3ステップが、**指一本** に圧縮される。

特に「チャンネル移動」を含められるのがデカい。わざわざマウスでサイドバーから探さんでええねん。

## 失敗談：やりすぎた「全自動化」

最初は調子に乗って、「朝の始業ボタン」を作ったんよ。
- Slack起動
- メール起動
- カレンダー起動
- Jira起動
- Notion起動
- Zoom起動

これ全部一気に立ち上がるボタン。名付けて「Morning Routine」。

**結果：Macが爆熱でファン全開になって固まった。**
あと、一気に情報が雪崩れ込んできて、朝イチから「うわぁ……」って鬱になった。

**【教訓】**
自動化は「必要な時に、必要なコンテキストだけ」呼ぶのが正解。
全部盛りは胃もたれするで。今は「Communication（Slack/Email）」と「Focus（Note/Jira）」でボタン分けてる。
