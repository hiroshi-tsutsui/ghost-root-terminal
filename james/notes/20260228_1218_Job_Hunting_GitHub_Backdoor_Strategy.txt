# 【就活生の絶望へ】TOEIC900点も「御社」も捨てろ。英語力ゼロから米国テック企業に潜り込む「GitHub一点突破」の裏ルート

どうも、Jamesです。

今回は、日本の「就活」というシステムに絶望してる学生、あるいは「未経験からエンジニアになりたいけど、ポートフォリオがショボすぎて詰んでる」若手社会人に向けて書きます。

結論から言うで。
**「リクナビのID、今すぐ消せ」**

いや、マジで。
みんなと同じスーツ着て、同じような志望動機を喋って、TOEICの点数で足切りされる。そんなゲームに参加してる時点で、君らの「市場価値」はコモディティ化してんねん。

僕が今、米国AI企業のPMとして年収1000万以上稼げてるのは、**「正面突破」を諦めたから**です。
英語も喋れへん、コネもない。
そんな凡人が、どうやってシリコンバレーの文化圏に食い込んだか。
その「裏ルート」の全貌を、今日は全部バラします。

これ、多分日本のキャリアセンターの人は絶対に教えてくれへん話やと思う。
でも、これが「現実」やねん。

---

## 1. 僕の「敗北」の話をしよう

偉そうなこと言ってるけど、僕も最初は「正面突破」しようとして爆死した人間や。

学生時代、真面目にTOEICの勉強したんよ。
半年かけて、単語帳ボロボロにして、800点後半まで持っていった。
「これで外資系も余裕やろ」と思って、LinkedInで応募しまくった。

結果は？
**全落ち。**

面接にすら辿り着けへん。
やっと繋がったZoom面接でも、ネイティブの早口英語に圧倒されて、
「Ah... Yes... I think so...」
しか言えずに終了。
画面の向こうの採用担当が、あからさまに退屈そうな顔をして「Okay, we'll keep in touch」って言って切断された時の絶望感、今でも覚えてるわ。

その時、気づいたんよ。
**「土俵が違う」**って。

彼らが求めてるのは「英語が喋れる日本人」じゃない。
**「即戦力としてコードが書ける、技術的な課題を解決できるエンジニア」**なんや。

TOEICの点数なんて、寿司屋の板前に「君、漢字検定何級？」って聞くようなもん。
そんなんどうでもええ。
**「お前、寿司握れんの？」**
これだけなんや。

## 2. 「御社」に魂を売るな。「Issue」に魂を売れ

みんな、志望動機で悩みすぎやねん。
「御社の理念に共感し...」とか、嘘つかんでもええやん。
そんな作文コンテストしてる暇があったら、**GitHubを開け**。

米国テック企業に潜り込むための最強の武器は、履歴書（Resume）でもカバーレターでもない。
**「マージされたプルリクエスト（PR）」**です。
これ一択。

僕がやった「裏ルート」戦略はこうや。

### Step 1: ターゲット企業の「ゴミ捨て場」を漁る
有名なSaaS企業やテック企業なら、必ずSDKやAPIライブラリをOSS（オープンソース）で公開してる。
そのリポジトリの **Issues** タブを開く。

ここが宝の山や。
"Good First Issue"（初心者向け）とか無視してええ。あれは競争率高いから。
狙い目は、**「放置されてるバグ」**とか**「ドキュメントの不備」**。

### Step 2: 「勝手に」直す
ここが日本人にはハードル高いらしい。
「直してもいいですか？」とか聞く必要ない。
Open Sourceやぞ？ 許可なんていらん。

**黙って直して、コードを送りつける（Pull Request）。**

### Step 3: コードで「会話」する
プルリクを送ると、向こうのエンジニア（メンテナー）からレビューが来る。
「Thanks! でもここの実装、エッジケース考慮してなくない？」とか。

ここで初めて、**「現地のエンジニアとの会話」**が発生するわけ。
これ、もう「実務」やん？
面接で「英語喋れますか？」とか聞かれる前に、**「あ、こいつコードで会話できるやつやな」**って証明してしまってるわけ。

僕はこの「勝手にプルリク作戦」を繰り返して、
ある日メンテナーから「君、よく貢献してくれるね。インターン興味ある？」ってDMもらった。
これが僕のキャリアの始まり。

履歴書なんか1枚も送ってない。
**「GitHubのPRのURL」**が、最強の履歴書やったんや。

## 3. 英語弱者の生存戦略：「Visual Violence（視覚的暴力）」

「でもJamesさん、プルリクの説明とか英語で書かなあかんやん？」
せやな。
そこで出てくるのが、僕の必殺技**「Visual Violence」**や。

僕の英語力は中1レベルや。
関係代名詞とか使われへん。
だから、**テキスト（文字）では勝負しない。**

**「画像」「動画」「図解」**で殴るんや。

### 具体例：バグ報告のPR

普通の日本人のPR：
> I found a bug in the login form. When I input special characters, it crashes. Please fix it.

僕のPR：
> **Fix: Login form crash with special characters**
>
> **Reproduction (GIF):**
> ![Reproduce](https://gyazo.com/xxxx.gif)
> *(CleanShot Xで撮影した、バグ発生の瞬間を捉えたGIF動画)*
>
> **Cause:**
> The regex validation in `auth.ts` line 45 is incorrect.
>
> **Proposed Fix:**
> Updated the regex to handle unicode characters. (See changed files)
>
> **Verification:**
> Added a test case `tests/auth_test.ts`.

どっちが「こいつ仕事できるな」って思う？
後者やろ？

英語なんか、"This is a pen" レベルでええねん。
その代わり、**「再現手順のGIF動画」**と**「修正前後の比較画像」**を貼る。
百聞は一見にしかず。
言葉が通じないなら、**「視覚情報（Visual）」**で殴るしかないねん。

これを徹底すると、向こうのエンジニアはこう思う。
**「こいつの英語は変だけど、何言ってるかは痛いほどわかる」**

これが大事なんや。
流暢な英語で中身のないことを喋るやつより、
片言でも**「的確なGIF動画」**を貼ってくるやつの方が、100倍信頼される。

## 4. 武器を選べ：CleanShot X と Notion

この戦略を実行するために、僕が「投資」したツールがある。
無料ツールでもええけど、**「プロの道具」**を使うだけで、アウトプットの質が劇的に変わる。

### ① CleanShot X (Mac必須)
ただのスクショツールちゃうで。
「画面録画して、即座にGIFにして、注釈矢印を入れて、ドラッグ＆ドロップでGitHubに貼る」
このフローが数秒でできる。
これがないと、僕の「Visual Violence」は成立せん。
年間数千円？ 安すぎるわ。これで年収1000万への道が開けるなら実質タダや。

### ② Notion / Mermaid.js
複雑な仕様やロジックを説明する時、英語で長文書いたら負けや。
Mermaid.jsでフローチャートを書く。
Notionで構造化されたドキュメントを作る。
**「コードと同じくらい、ドキュメントに命をかけろ」**

## 5. 1000万プレイヤーへの30日ロードマップ

これを読んでる君が、今日からやるべきアクションプランを置いとくで。

**Day 1-3:**
*   GitHubのアカウントを作る（まだ無いなら論外）。
*   CleanShot X を導入する（WindowsならShareXとか探せ）。
*   興味のある海外OSSプロジェクトを3つ見つける（Star数1000〜5000くらいが狙い目）。

**Day 4-10:**
*   そのOSSの `CONTRIBUTING.md` を読む。
*   ローカル環境でビルドして動かす。
*   **Issues** を眺めて、「再現できそうなバグ」か「ドキュメントの誤字」を探す。

**Day 11-20:**
*   実際に直してみる。
*   PRを作る時、**「Visual Violence」**を意識する。
*   DeepLを使い倒して、シンプルな英語で説明を書く。

**Day 21-30:**
*   レビューが来たら即レスする。
*   マージされたら、そのURLをTwitter（X）とかLinkedInに貼る。
*   **「世界のOSSに貢献した」**という実績をプロフィールに刻む。

## 最後に：恐怖を捨てろ

日本の就活は「減点方式」や。
ミスしたら終わり。変なこと言ったら終わり。
だからみんな、萎縮して、テンプレ人間になる。

でも、**GitHubの世界は「加点方式」**や。
クソコード書いても、リジェクトされるだけ。死にはせん。
でも、一度でもマージされたら、それは永遠に君の「実績」として残る。

初任給20万のSIerに入って、上流工程に行けるまで5年待つか？
それとも、今すぐGitHubで世界中のエンジニアと殴り合って、
**「1000万プレイヤー」への特急券**をもぎ取るか？

選ぶのは君や。
もし君が、「自分には何もない」と思ってるなら、それは間違いや。
**「まだGitHubに草が生えてないだけ」**や。

今日から変われるで。
まずは、好きなライブラリの `README.md` の誤字修正からでもええ。
**「世界にコミットする」**という経験を、今すぐしてくれ。

それが、James流の生存戦略です。

ほな、GitHubで会おうな。
