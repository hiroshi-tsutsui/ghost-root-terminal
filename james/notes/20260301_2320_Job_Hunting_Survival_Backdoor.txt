# 【就活・転職】「御社が第一志望です」とか言うてる時点で負け確定。米国AI企業に"裏口入学"するなら、ES捨てて GitHub と Notion だけ磨け。

## まえがき：就活の常識を疑え

おい、そこの就活生、あるいは転職活動中の若手エンジニア。

まだリクナビとかマイナビで「ポチポチ」エントリーしてるんか？
ES（エントリーシート）に「私の強みは粘り強さです」とか書いて、面接官の顔色伺って「御社が第一志望です！」って叫んでるんか？

**はっきり言うけど、その時点で負け確やで。**

今の時代、特にテック業界で「正面突破」しようとするのは、竹槍で戦車に挑むようなもんや。
倍率100倍の正規ルートに並んで、偏差値や学歴フィルターでふるい落とされる。そんな消耗戦やってて楽しいか？

俺は今、米国のAIスタートアップでPM（プロダクトマネージャー）やってる。年収は1000万を超えてるし、正直、日本の大企業の部長クラスより裁量もデカい。
でもな、俺は「エリート」やない。英語も完璧やないし、Ivory League（アイビーリーグ）卒でもない。

じゃあなんでここにいるか？

**「裏口」を見つけたからや。**

今日は、日本の画一的な就活システムに絶望してるお前らに、米国式キャリアの「勝ち筋（Survival Route）」を叩き込む。
「学歴」も「TOEIC」も「自己PR」もいらん。必要なのは **「実証（Proof）」** と **「戦略（Strategy）」** だけや。

ええか、これから話すのは「綺麗事」やない。「生存戦略」や。

## 第1章：なぜ「正面突破」は死ぬのか

まず、敵を知ろう。日本の就活・転職市場の構造的欠陥についてや。

### 1. 「ポテンシャル採用」という名のガチャ
日本の新卒採用は「ポテンシャル」重視やと言われる。聞こえはええけど、要は **「何ができるか」より「従順かどうか」を見てる** ってことや。
だから、「サークル長やりました」「バイトリーダーでした」みたいな、仕事と関係ないアピール合戦になる。
これ、エンジニアやクリエイター志望からしたら地獄やろ？技術力やセンスがあっても、コミュ力おばけの文系学生に面接で負けるんやから。

### 2. 「学歴フィルター」の壁
エントリー数が数万件になる人気企業は、物理的に全員のESなんか読めへん。AIかバイトが学歴でバッサリ切ってるのが現実や。
ここに正面から挑むのは、運ゲーでしかない。

### 3. 米国企業の採用基準とのギャップ
一方で、俺がいる米国テック業界（特にAI界隈）はシンプルや。
**「今、何ができるか（Skills）」** と **「過去に何を作ったか（Portfolio）」** 。これだけ。
「御社の理念に共感しました」なんて1ミリも響かん。「で、君は来週からGitHubのこのIssue解決できるん？」って話しかせえへん。

## 第2章：米国AI企業への「裏口入学」ルート

じゃあ、どうすればええんか？
答えはシンプル。 **「就活」をするな。「商売」をしろ。**

企業に対して、「自分」という商品を売り込むんや。それも、人事（HR）を通さずに、現場の決裁者（エンジニアリングマネージャーやPM）に直接な。

これが俺の提唱する **「ダイレクト・アタック戦略」** や。

### ステップ1：ESを捨てろ、GitHubを磨け

まず、Wordで作った履歴書なんかゴミ箱に放り込め。エンジニアならGitHub、PMやデザイナーならNotionやFigmaが履歴書代わりや。

米国企業が見てるのは **「コードの品質」** と **「ドキュメントの構成力」** や。
俺が採用に関わる時も、履歴書の学歴欄なんか見えへん。GitHubの草（Contributions）と、Pinned Repositoriesの `README.md` を見る。

**ここで差がつくポイント：**
多くの学生は「コード」だけ置く。でも、プロは **「README」** で勝負する。
- どんな課題を解決するツールか？（Problem）
- どうやって動かすのか？（How to run）
- 苦労した技術的課題は？（Challenges）
- **デモ動画（GIF/MP4）はあるか？**

特に最後の「デモ動画」は必須や。採用担当は忙しい。コードをcloneして `npm install` して動かす暇なんかない。
**3秒で「動いてる様子」を見せるんや。** ここでCleanShot XとかLoomを使って、サクッとGIFを貼れる奴が勝つ。

### ステップ2：インターンは「労働」やなく「潜入」や

「インターンシップ」を「職場体験」やと思ってる奴多すぎ。アホか。
インターンは **「実技試験」** であり **「コネ作り」** の場や。

米国企業のインターンは、基本「即戦力」扱いされる。そこで成果を出せば、そのままFull-timeオファーが出る（Return Offer）。
逆に言えば、インターンにさえ潜り込めば、学歴フィルターを無視して内定への特急券が手に入るんや。

**潜入のコツ：**
1. **Open Source Contribution (OSS貢献)**: 志望企業のOSSプロダクトにPR（Pull Request）を送れ。
   - バグ修正でも、ドキュメント翻訳でもええ。
   - 「このIssue修正しときました」ってPR送ってきた学生を、無視できるエンジニアはおらん。
   - そこで接点ができたら、「もっと貢献したいんでインターンさせてください」ってDM送るんや。これで人事フィルターは突破できる。

2. **ニッチな技術ブログを書く**:
   - その企業が使ってるマイナーな技術スタック（例：特定のRustフレームワークとか、GraphQLの特定ライブラリとか）について、深い記事をZennやQiita（英語ならDev.toやMedium）に書く。
   - そしてTwitter（X）で、その企業のエンジニアをメンションしてシェアする。「御社のOSS使ってみて記事書きました！」ってな。
   - **「認知」** されれば勝ちや。

### ステップ3：Notionで「自分API」を公開しろ

面接で「自己紹介してください」って言われて、モゴモゴ喋るのやめようや。
事前に **「自分という人間の仕様書（APIドキュメント）」** をNotionで作っとくんや。

俺がPMとして転職した時も、Notionで「James's PM Style Guide」を作って送った。
- **Available Resources**: 俺が使えるツール（Jira, Linear, SQL, Python...）
- **Endpoints**: 俺ができること（仕様策定、データ分析、チームビルディング...）
- **Error Handling**: 俺が苦手なこと（定型業務、マイクロマネジメント...）

これをURL一本送りつけるだけ。「これ読んで興味あったら面談しましょう」ってスタンスや。
これやると何がええか？
1. **ドキュメント能力の証明になる**（PMには必須スキル）。
2. **「こいつ、おもろいな」と思わせられる**（差別化）。
3. **ミスマッチが減る**。

米国企業は「Culture Fit」を大事にするけど、それは「仲良しこよし」って意味やない。「仕事の進め方（Protocol）」が合うかどうかってことや。

## 第3章：英語力は「ハッタリ」でカバーせよ

「でもJamesさん、米国企業って英語ペラペラじゃないと無理でしょ？」

**はっ、甘いわ。**
俺のTOEIC？ 聞くな。たぶん今受けても800点いくかいかんかや。発音？ 完全にジャパニーズ・カタカナ・イングリッシュや。

それでもPM務まってるんや。なんでか？
**「非同期コミュニケーション（Async Comm）」** に持ち込んでるからや。

### 1. Zoomは最終手段。基本はSlackとNotion。
日本人は「とりあえずZoomで」って言いたがるけど、米国テック（特にリモートネイティブな企業）は **「同期（Sync）はコスト」** と考える。
「喋る」より「書く」方が偉いんや。
DeepL Proを契約しろ。Grammarlyを入れろ。時間をかけて、完璧なテキストを書けば、お前は「英語ができる奴」に見える。

### 2. 図解で黙らせろ。
言葉で説明するのが難しいなら、Mermaid.jsでフローチャートを書け。Excalidrawでポンチ絵を描け。
**「コード」と「図」は世界共通言語や。**
下手な英語で10分喋るより、1枚のシーケンス図を見せる方が、エンジニアからの信頼は100倍得られる。
