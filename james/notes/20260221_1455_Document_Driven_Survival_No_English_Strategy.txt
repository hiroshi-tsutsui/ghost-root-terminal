# 【実話】「英語力」を捨てて「年収1000万」を掴んだ、ドキュメント駆動の生存戦略

## 0. はじめに：英語ができるから勝てるんちゃう。

「Jamesさん、やっぱ米国企業のPMやるなら、英語ペラペラなんですよね？」

これ、週に3回は聞かれるんやけど。
結論から言うわ。**僕の英語力、マジで中学生レベルやで。**

TOEICの点数は確かに975点ある。でもな、実際の会議でネイティブのマシンガントーク聞き取れるか？ 答えはNOや。
映画を字幕なしで見れるか？ 全く無理。
カフェで雑談できるか？ 愛想笑いで逃げるレベル。

それでも、**僕は米国AI企業のPMとして、年収1000万超えのポジションで生き残っとる。**
なぜか？

それは、僕が**「英語力」という土俵で戦うのを辞めたから**や。

多くの日本人は勘違いしてる。「英語さえできれば、グローバルで活躍できる」と。
ちゃうねん。逆や。
**「英語ができない」という弱点を、「ドキュメント」という最強の武器で補完する**ことで、逆にネイティブよりも信頼されるポジションを築けるんや。

今日は、凡人が「英語」というハンデを背負ったまま、米国テック企業の最前線で勝ち抜くための**「ドキュメント駆動生存戦略（Document Driven Survival）」**を全部話す。

これ読んだら、明日からのZoom会議が怖くなくなるはずや。

---

## 1. 音声コミュニケーションは「死」やと思え

まず、前提を破壊するで。
日本人は真面目やから、「会議で発言しなきゃ」とか「リスニング力上げなきゃ」って必死に英会話スクール通うやろ？

**その努力、ROI（投資対効果）悪すぎるで。**

ネイティブが30年かけて培った言語感覚に、大人になってから挑んでも勝てるわけないやん。
彼らの土俵（音声・リアルタイム・アドリブ）で戦ったら、負けるに決まっとる。

### 米国テック企業の会議、こんなんやぞ
実際、僕が入社して最初の会議はどうやったかと言うと…地獄やった。
Zoomに入った瞬間、CTOが早口で「Blah blah blah architecture blah blah scale!」ってまくし立てる。
他のメンバーも「Yeah, totally agree. But what about X?」って即座に被せてくる。
**僕が口を挟む隙間なんて、1ミリもない。**

しかも、彼らはスラングや独特の言い回しを多用する。「Let's table that for now（それは一旦置いとこう）」とか「Low hanging fruit（すぐ取れる成果）」とか。
辞書にも載ってへん表現が飛び交う中で、僕はただ「へらへら笑って頷くだけの置物」やった。
**悔しかったで。** 日本ではバリバリ仕事してたのに、ここでは完全に無力な自分が情けなくて、会議の後トイレで泣きそうになったこともある。

でもな、そこで気づいたんや。
**「こいつら、喋ってるだけで、何も決まってへんやん」**と。

そう、彼らは議論が大好きやけど、議論を発散させるだけで、収束させるのが苦手なんや。
「いいアイデア出たね！」で終わって、結局誰がいつまでに何をするかが曖昧なまま解散することが多い。

ここに、僕ら日本人が勝てる隙間がある。
土俵を変えるんや。
**「音声（Synchronous）」から「文字・図解（Asynchronous）」へ。**

僕が徹底してるルールはこれや。
**「口頭で決まったことは、存在しないものとする」**

会議で「OK, let's do that!」ってなっても、議事録に残ってなければ「やってない」と同じ。
逆に言えば、**「ドキュメントに書いたもん勝ち」**の世界なんよ、米国テック企業は。

英語が聞き取れなくても、喋れなくても、**「正確な仕様書」**と**「美しい図解」**があれば、PMは務まる。
むしろ、喋りだけで適当なこと言うネイティブPMより、テキストで確実な仕事をする僕の方が信頼される。

これが、僕の生存戦略の核心や。

## 2. 武器を持て：「DeepL」じゃなくて「Notion」と「CleanShot X」

じゃあ具体的にどうするか。
ここで大事なのは、**「英語力」ではなく「ツール力」**や。

僕が使ってる三種の神器を紹介する。これだけで戦闘力5倍になるで。

### ① Notion（思考のOS）
会議の前に、**「アジェンダ」を完璧に作り込む**。
何を決める会議なのか、論点は何か、A案とB案のメリット・デメリットは何か。
これを全部Notionに書いて、会議の24時間前に共有するんや。

会議が始まったら、そのNotionを画面共有する。
**「英語」を聞かせるんじゃなくて、「文字」を読ませる。**
これで、議論の主導権は完全に僕のものになる。
彼らは僕の英語じゃなくて、僕のNotionを見て会話せざるを得ないからな。

さらに言うとな、**Notionは「非同期コミュニケーション」の神**や。
時差があるチームとも、Notionにコメントを残しておけば議論が進む。
僕が寝てる間に、USチームがコメントを返してくれる。
起きたら、議論が終わってたりする。最高やろ？

### ② CleanShot X（視覚の暴力）
Macユーザーなら絶対に入れろ。これがないと仕事にならん。
スクリーンショット、GIF録画、注釈入れが爆速でできる神ツールや。

バグ報告や仕様説明で、ダラダラ英語のテキスト打ってる暇があったら、**CleanShot XでGIF撮って貼れ。**
「百聞は一見にしかず」はガチや。
複雑なUIの挙動も、3秒のGIFなら一発で伝わる。
**英語で説明するより、動画見せた方が100倍速いし、誤解がない。**
これを僕は**「Visual Violence（視覚の暴力）」**と呼んでる。有無を言わせぬ説得力や。

例えば、「ボタンを押したらメニューが出るはずが出ない」というバグ報告。
英語で書くと：
"When I click the button, the menu is supposed to appear, but it doesn't."
これでも通じるけど、エンジニアは再現確認しなアカン。
でもGIFがあれば、一発で「あ、これね」ってなる。
**エンジニアの脳内コストを下げること**こそが、PMの最大の価値提供や。

### ③ Linear（開発の神）
エンジニアとの会話は、チャットじゃなくてチケット（Linear）でやる。
チケットには、要件、デザイン（Figma）、再現手順（GIF）を全部詰め込む。
エンジニアは英語の長文なんて読みたくない。
**「何を作ればいいか」が明確なチケット**があれば、彼らはハッピーなんや。
PMの仕事は、エンジニアを迷わせないこと。英語で励ますことちゃうで。

Jira使ってる会社もあるけど、Linearの爆速動作に慣れたら戻れへん。
ショートカットキーだけでタスクが捌ける快感。
**ツールへのこだわりは、仕事へのこだわりそのものや。**

## 3. 実践編：会議で沈黙しても「勝つ」方法

「でもJamesさん、会議で地蔵（無言）になるのは怖いです…」

わかる。怖いよな。
でもな、**「無言」と「無能」は違う。**

会議中に喋らなくても、**会議後に「決定事項」と「ネクストアクション」を即座にSlackに投げる**やつが最強なんや。

### 僕の会議ハック（The "Recap" Strategy）
1. 会議中は、ひたすらメモを取る（Notion）。
2. 聞き取れなかったら、**「Sorry, could you rephrase that for the notes?（議事録のために言い直して）」**って言う。これなら「英語できない」じゃなくて「真面目な書記」に見える。
3. 会議終了後、**5分以内**に「Recap（まとめ）」をSlackに投げる。

```
Summary of our discussion:
- Decision: Option A is selected because of X.
- Next Steps:
  - @EngineerA: Implement backend API by Friday.
  - @James: Update spec doc by tomorrow.
```

これをやるだけで、**「会議を仕切ったのはJames」という事実が確定する。**
喋ってなくても、アウトプット（議事録）を出したやつが正義なんや。

これができれば、英語の発音なんてどうでもええ。
**「Jamesの会議は結論が出る」「Jamesがいればプロジェクトが進む」**
この信頼さえ勝ち取れば、年収1000万なんて通過点や。

さらに言うと、**「非同期で決める」**という文化を作ることにも繋がる。
会議で決めきれなかったことは、「Slackで続きやろう」って誘導する。
Slackなら翻訳ツール使い放題やし、じっくり推敲して返信できる。
**自分の得意なフィールド（テキストコミュニケーション）に引きずり込むんや。**

## 4. なぜ「年収1000万」なのか？

ちょっと現実的な話をしようか。
なんで僕がここまでして、米国企業のPMにしがみついとるか。
それは、**リターンが桁違い**やからや。

日本のIT企業でPMやってても、年収600〜800万くらいで頭打ちになることが多い。
でも、米国テック企業なら、ジュニアレベルでも軽く1000万超える。
シニアになれば、2000万、3000万の世界や。

**同じ仕事をしてるのに、給料が2倍、3倍違う。**
これ、バグってると思わへんか？
でも、これが現実なんや。

英語ができないからって、このチャンスを逃すのはあまりにも勿体無い。
「英語力」という幻想に縛られて、自分の市場価値を低く見積もってないか？
**技術力、推進力、論理的思考力。**
日本人が持ってるポテンシャルは、世界でも十分に通用する。
ただ、「言葉の壁」をどう乗り越えるか、その戦術を知らんだけなんや。

だからこそ、僕は声を大にして言いたい。
**ドキュメントを武器にしろ。ツールを使い倒せ。**
そうすれば、君もこのフィールドに立てる。

## 5. 結論：英語コンプレックスを捨てろ、武器を磨け

最後に言いたい。
もし今、「英語ができないから」って海外挑戦を諦めてるエンジニアやPMがおるなら、今すぐその考えを捨てろ。

英語なんて、ただのツールや。
しかも、AI（DeepLやChatGPT）のおかげで、言語の壁はどんどん低くなっとる。

重要なのは、**「何を伝えるか」**というコンテンツと、**「どう伝えるか」**というデリバリー能力や。
ドキュメント力、図解力、ツール使いこなし力。
こっちの方が、ネイティブ並みの発音よりよっぽど希少価値が高い。

**「英語が下手な日本人」**で終わるな。
**「英語は下手やけど、仕事が爆速で正確なプロフェッショナル」**になれ。

それが、僕らがグローバル市場で生き残るための、唯一にして最強の生存戦略や。
