# 【実話】「英語力」よりも「図解力」。TOEIC 500点台の僕が米国AI企業のPMとして生存している「沈黙のドキュメント戦術」全公開

まいど、Jamesです。

今回は、ちょっと「耳の痛い」話をします。
特に、「英語が完璧になってから海外転職しよう」とか思ってる人。
あるいは、「Zoom会議でネイティブの議論についていけなくて死にたい」と思ってる現役の人。

結論から言うで。
**「英語の勉強」なんか今すぐやめろ。**
**その代わり、「スクリーンショット」と「矢印」の使い方を極めろ。**

今日は、TOEIC 975点（今はな）とか偉そうに書いてる僕が、実は渡米当初 **「Listen? No. Speak? No. Code? Yes.」** みたいな絶望的な英語力やったのに、なぜクビにならずに、むしろ「Jamesの言うことは絶対だ」というポジションを築けたんか。

その泥臭い「生存戦略」の全てを公開します。
これは英語の話ちゃうで。「コミュニケーションのハック」の話や。

## 1. 「英語ができる」の定義が間違ってる

まず、大前提として日本人が勘違いしてること。
「英語ができる」＝「流暢にペラペラ喋れること」やと思ってるやろ？

これ、**大間違い** やで。

米国テック企業、特にAI開発の現場において「コミュニケーションができる」の定義はこれだけや。
**「認識の齟齬（そご）なく、仕様と納期を合意できること」。**

以上。
ここに「発音の美しさ」も「文法の正確さ」も1ミリも入ってない。

僕が最初に配属されたチームは、インド人、中国人、ロシア人、そしてアメリカ人の混成チームやった。
全員、クセが強い。インド人の早口英語なんか、最初は「呪文」にしか聞こえへんかった。

会議中、僕は貝のように黙ってた。「Yes... Yes...」って相槌打つだけの置物。
**完全に「クビ候補」筆頭やった。**

でもある日、気づいたんや。
ネイティブ同士でも、実は「話が通じてない」ことが多いってことに。

Aさん「このAPIのレスポンスはJSONで返すべきだ」
Bさん「いや、そこはProtocol Buffersでしょ」
Aさん「え、でもフロントエンドが...」

こんな議論が30分続いて、結局何も決まらん。
ここで僕は、ある「武器」を取り出した。

## 2. 武器は「CleanShot X」と「Mermaid.js」

僕が何をしたか。
黙って画面共有を奪い取って、1枚の図を見せたんや。

そこには、複雑な英文は一切ない。
四角い箱（Box）と、矢印（Arrow）だけ。

```mermaid
sequenceDiagram
    participant Front as Frontend
    participant API as API Server
    participant DB as Database
    
    Front->>API: GET /users/123 (JSON)
    Note right of API: Here is the point!
    API->>DB: Query
    DB-->>API: Result
    API-->>Front: { "id": 123, "proto": "bin" } ???
```

その図の「???」の部分を無言で指差して、チャットに一言だけ書いた。
**"Which format here? JSON or Proto?"**

その瞬間、30分の議論が凍りついた。
全員が画面を凝視して、Aさんが言った。
"Oh, JSON right there."

Bさんも頷いた。
"Agreed. Let's go with JSON for now."

**会議終了。**

この時、僕は確信した。
**「流暢な英語」よりも、「正確な図解」の方が、100倍強い。**

米国企業において、PM（プロダクトマネージャー）に求められるのは「演説」ちゃうねん。「交通整理」やねん。
複雑な事象を、誰が見ても誤解しようがない「図」に落とし込める能力。
これさえあれば、中学英語で十分戦える。

## 3. James流「沈黙のドキュメント」3種の神器

ほな、具体的にどうすればええんか。
僕が毎日使ってる「武器」を紹介するで。これを使いこなせば、今日から君も「喋らないPM」になれる。

### ① CleanShot X（Mac最強のスクショツール）

Mac標準のスクショ（Cmd+Shift+4）使ってる人、今すぐやめなはれ。時間の無駄や。
CleanShot Xは、撮った瞬間に「注釈（Annotation）」が入れられる。

- **矢印（Arrow）**: 視線の誘導。
- **枠囲み（Box）**: 注目ポイントの明示。
- **カウンター（Counter）**: 手順（①、②、③...）の付与。
- **モザイク（Pixelate）**: 機密情報の隠蔽。

バグ報告なんか、文章で書く必要ない。「期待値」と「現状」をスクショに書き込んで、Slackにペタッ。
**"Fix this to look like that."**
これだけで通じる。エンジニアも「Oh, got it.」で終わり。
お互いにハッピーやろ？

### ② Mermaid.js（テキストで書く図解）

さっき見せたシーケンス図みたいなやつな。
NotionとかGitHubのMarkdownで直接書けるのが最強や。

PowerPointとかFigmaでチマチマ図を描くな。修正が面倒くさい。
コード（テキスト）で図を書けば、仕様変更があった時もテキストを書き換えるだけ。
Gitでバージョン管理もできる。

**「図をコードで管理する」**。これがエンジニア出身PMの最強の生存戦略や。

### ③ DeepL & Grammarly（AIによる武装）

これはもう説明不要かもしれんけど、使い方にコツがある。
DeepLで翻訳した英文を、そのまま貼るな。
**必ず「短く」しろ。**

DeepL: "I think it would be better if we could consider changing the color of this button to red."
↓
James修正: **"Change button color to Red. Reason: Visibility."**

米国人はせっかちや。丁寧語なんか要らん。
**Command（命令）とReason（理由）。** この2つがあれば動く。
Grammarlyでスペルミスだけ潰して、あとは箇条書きで投げろ。

## 4. 「英語ができない」を「アドバンテージ」に変える

ここまで読んでも、「でもやっぱり会議で発言できないのは怖い...」って思うかもしれん。

逆転の発想を持て。
**「英語が苦手だからこそ、ドキュメントに残す」というキャラを確立するんや。**

僕は最初にチームにこう宣言した。
**"My English is not perfect. So I write everything down on Notion to avoid misunderstanding. Please check the doc."**
（英語完璧ちゃうから、誤解避けるために全部Notionに書くわ。ドキュメント見てな。）

これを言うと、周りはどう思うか？
「こいつ、英語できない無能」じゃなくて、**「こいつ、めちゃくちゃ誠実で慎重なPM」** という評価になるんや。

実際、口頭だけで「言った言わない」のトラブルになるPMより、
「全部ドキュメントに書いてあるJames」の方が、エンジニアからの信頼は圧倒的に厚い。
なぜなら、エンジニアもまた、曖昧な口頭指示より、明確な仕様書（ドキュメント）を好む生き物やからな。

### さらに「非同期」という最強の武器を手に入れる

そして、ここからが重要や。
ドキュメントに頼るということは、必然的に**「非同期コミュニケーション（Asynchronous Communication）」**に移行するということや。

これが何を意味するか分かるか？
**「時差」と「即レス圧力」からの解放や。**

米国企業、特にシリコンバレーのテック企業は、世界中にメンバーが分散している。
「Zoom会議」は、全員の時間を強制的に拘束する、実は「コストの高い」行為なんや。

そこで、僕の戦術がハマる。
1. 会議前に完璧なドキュメント（図解入り）をSlackに投げる。
2. **"Please read this before the meeting. Comment if any questions."** と一言添える。
3. メンバーが事前に読む。コメントがつく。
4. 会議が始まる頃には、議論の8割が終わっている。
5. 会議時間は15分で終わる。**"Looks good. Let's do it."** で解散。

これ、英語ができない僕が「生き残るため」に編み出した苦肉の策やった。
でも結果的に、**「Jamesの会議は短くて生産的だ」** という評価に繋がったんや。

ネイティブのPMは、喋るのが大好きやから、どうしても会議が長引く。
雑談も多いし、議論が発散する。
でも僕は、言葉のハンデがあるからこそ、事前に完璧に準備し、会議を最短で終わらせることに命をかけてる。

これが、僕が米国企業で重宝される理由や。
**「英語力」の欠如が、「生産性」という別の価値を生んだんや。**

だから自信を持ってほしい。
君の「英語コンプレックス」は、使い方次第で「最強の武器」になる。
流暢に喋れないなら、喋らなくていい仕組みを作ればいい。
それがPM（プロダクトマネージャー）の仕事そのものやからな。

## 5. 結論：言葉を捨てて、図を描け

日本人は真面目やから、どうしても「英語のハンデ」を「英語の勉強」で埋めようとする。
でもな、ネイティブの語彙力やスピードに追いつくには10年かかる。
そんな暇ないやろ？ 明日の会議どうすんねん。

**今すぐ「言葉」を捨てろ。**
**「図」と「矢印」と「スクリーンショット」で戦え。**

それが、僕ら「非ネイティブ」が、シリコンバレーの猛者たちと互角以上に渡り合うための、唯一にして最強の「裏ルート」や。

英語の勉強してる暇があったら、CleanShot X 課金して、ショートカット手になじませろ。
Mermaid.js の書き方覚えろ。

それが、年収1000万への最短距離やで。

ほな、また。
