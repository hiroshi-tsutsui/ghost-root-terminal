# 【脱・タブ破産】Arc Browserの「多重人格運用」で、米国PMの脳内メモリを物理的に守る話

正直に言うてええか？ お前らのブラウザ、汚すぎん？

いや、人のこと言えんかったんよ、俺も。
米国テック企業のPMやってると、開かなあかんタブの量が異常やねん。
JIRAのチケット、Figmaのデザイン、PRのGitHub、Notionの仕様書、Slackのスレッド、競合調査の別タブ...。

気づいたらChromeのタブが「米粒」みたいになってて、どれがどれかわからん。
**「あれ、さっき見てたGoogle Docどこいった？」** って探すだけで、1日合計30分は無駄にしてた自信あるわ。

これ、「探す時間」が無駄なだけちゃうねん。
**「探してる間に別の通知が目に入って、集中力が切れる」** のが一番の損失やねん。

で、結論から言うとな。
**「Arc Browser」に変えて、運用ルールをガチガチに固めたら、脳のメモリ消費が半分になった。**

ただ、Arc入れて「わーい、見た目がオシャレ！」で終わってるやつ多すぎ。
それじゃ意味ないねん。
今回は、俺が試行錯誤の末にたどり着いた**「多重人格ブラウジング」**の設定を全部晒すわ。

## 1. なぜ Chrome じゃダメなのか（失敗の歴史）

最初はChromeの「タブグループ」機能使ってたんよ。
「Work」とか「Private」とか色分けしてな。

でもな、あれ**「結局同じウィンドウ内にノイズがある」**のがアカンねん。
JIRA見てる時に、横のタブにYouTubeのアイコン見えたら、人間誰しもクリックしたくなるやろ？

あと、別プロジェクトのアカウント切り替えな。
「あれ？今どっちのGoogleアカウントでログインしてるっけ？」ってなって、権限エラーでイライラするやつ。あれで寿命縮むわ。

## 2. James流「多重人格セットアップ」

Arcの真髄は「Spaces（スペース）」と「Profiles（プロファイル）」の掛け合わせにある。
俺はこれを**「脳のモード切り替えスイッチ」**として使ってる。

### Step 1: プロファイルを物理的に分ける

これが一番重要。
Arcの `Settings > Profiles` から、以下の3つを作る。

1.  **Work (Main)**: 会社のGoogleアカウントに紐付け。
2.  **Dev / Research**: 検証用アカウント。Cookieも拡張機能もMainとは別。
3.  **Personal**: プライベート用。NetflixとかSNSは全部こっち。

ここで大事なのは、**「Workプロファイルには、YouTubeやSNSのログイン情報を一切入れない」**こと。
仕事中に魔が差してTwitter見ようとしても、ログイン画面が出てきて「ハッ」と我に返る。
この**「物理的な障壁」**を作るのが、意志の弱い俺らには必要やねん。

### Step 2: Spaceで「文脈」を固定する

プロファイルを作ったら、次は「Space（スペース）」や。
サイドバーをスワイプするだけで切り替わるアレな。
俺はこう分けてる。

*   **🚀 Execution (Work Profile)**
    *   ここには「今、手を動かすもの」しか置かない。
    *   Pinned Tabs: JIRA, GitHub, Figma, Linear
    *   **ルール: ここで調べ物はしない。**

*   **🧠 Thinking (Work Profile)**
    *   ここは「考える」場所。
    *   Pinned Tabs: Notion, Google Docs, Miro, ChatGPT (Team)
    *   **ルール: 通知が来る系（Slack/Mail）は置かない。**

*   **☕ Chill (Personal Profile)**
    *   休憩用。
    *   Pinned Tabs: YouTube, Twitter, Gmail (Personal)

こうやって**「作業の種類」ごとに部屋を分ける**んよ。
「今日はドキュメント書く日や！」と思ったら、`Thinking` スペースに引きこもる。
そこにはJIRAのチケットもSlackの通知も見えへん。これだけで集中力が段違いや。

## 3. 失敗談：Spacesを細かくしすぎるな

導入初期、俺はイキってこんな分け方をしてた。

*   Space 1: Project A
*   Space 2: Project B
*   Space 3: Project C
*   Space 4: Admin
*   Space 5: HR...

**これ、大失敗やった。**
プロジェクトAの資料見ながらプロジェクトBのチャット返す、みたいな時に、指が腱鞘炎になるぐらいスワイプせなあかん。
コンテキストスイッチのコストが高すぎて、逆に効率落ちたわ。

**教訓:**
Spaceは「プロジェクト単位」じゃなくて**「脳のモード（作業・思考・休憩）」**で分けるのが正解。

## 4. 神機能「Air Traffic Control」を設定せよ

ここまで設定しても、Slackのリンク踏んだら勝手に変なSpaceで開いたりするやろ？
それを防ぐのが **「Air Traffic Control」** や。
(`Settings > Links > Air Traffic Control`)

これが地味に最強。特定のURLパターンを、特定のSpaceに強制的に飛ばせる。

*   `figma.com` を含むURL → **🚀 Execution** Space で開く
*   `notion.so` を含むURL → **🧠 Thinking** Space で開く
*   `youtube.com` を含むURL → **☕ Chill** Space で開く

これを設定しておくと、Slackで「このFigma確認して！」ってリンク踏んだ瞬間、
勝手に `Execution` Space に画面が切り替わってFigmaが開く。

**「脳が準備する前に、ツールが勝手に環境を整えてくれる」** 感覚。
これ味わったら、もう手動でタブ整理とかアホらしくてやってられんで。

## まとめ：ブラウザは「道具箱」やない、「作業部屋」や

Chromeは「なんでも放り込める巨大な道具箱」やった。
でもArcは、目的ごとに整理された「作業部屋」を作れる。

*   **プロファイルで「公私」を物理遮断**
*   **Spaceで「思考モード」を切り替え**
*   **Air Traffic Controlで「動線」を自動化**

この3つやるだけで、夕方の脳の疲れ方が全然ちゃうで。
だまされたと思って、今すぐ設定してみて。
最初は「慣れへんわ！」ってなるかもしれんけど、3日で戻れんくなるから。

ほな、俺は `Thinking` Space に戻って仕様書書くわ。またな。
