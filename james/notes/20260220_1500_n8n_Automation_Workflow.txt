## そもそも、n8nって何なん？

一言で言うと、**「ノードベースのワークフロー自動化ツール」**。
ZapierとかMake（旧Integromat）の親戚やけど、決定的な違いがある。

1.  **Self-hosted（自前サーバー）なら無料:** 自分のMacとかVPS（Docker）で動かせば、タスク数無制限でタダ。
2.  **自由度が異常:** JavaScript書けば何でもできる。「もしAならB、そうでなければC」みたいな分岐も余裕。
3.  **UIがイケてる:** 繋いでいくのがパズルみたいで楽しい。

ワイは最初、家のラズパイで動かしてたけど、今は安定稼働のために月5ドルの激安VPSで回してる。Zapierに月30ドル払うより断然安いし、勉強にもなるし一石二鳥や。

## James流：競合監視・要約ボットの作り方

今回紹介するのは、以下の流れを自動化するワークフローや。

1.  **Trigger:** 毎日朝7時に起動（Cron）。
2.  **Get RSS:** 競合他社（3社）のテックブログのRSSを取得。
3.  **Filter:** 「昨日更新された記事」だけを抽出。
4.  **Summarize (AI):** 記事の中身をスクレイピングして、LLM（OpenAIとかClaude）に投げて**「3行で要約」**させる。
5.  **Notify:** Slackの `#competitor-watch` チャンネルに投稿。

これ、Zapierでやろうとしたら「Multi-step Zaps」が必要で有料プラン確定や。
n8nなら、これが**タダ**で組める。最高かよ。

### Step 1: Cronノードで「朝のリズム」を作る

n8nのキャンバスに `Schedule` ノードを置く。
設定は `Every Day` at `07:00`。
これで、ワイが起きてコーヒー淹れてる間に、ボットが動き出すわけや。

### Step 2: RSS Feed Readノードで情報を吸い上げる

次に `RSS Feed Read` ノードを繋ぐ。
URLには競合のブログURLを入れるんやけど、ここで**失敗談**がある。

**【失敗談①：全部取ってきてパンク】**
最初はフィルタリングせずに「最新10件」とか全部Slackに流しててん。
そしたらSlackが通知の嵐になって、チームメンバーから「James、これスパムやで」って怒られたわ。
**「情報の価値は、量じゃなくて質」**やな。反省。

### Step 3: Functionノードで「昨日」に絞る

ここがn8nの真骨頂。JavaScriptでロジックが書ける。
「記事の公開日時」が「昨日の0:00〜23:59」に含まれるか判定するコードを書く。

```javascript
// まぁこんな感じのイメージや（AIに書かせたら一瞬やで）
const yesterday = new Date();
yesterday.setDate(yesterday.getDate() - 1);
// ...日付比較ロジック...
return items.filter(item => item.date > yesterday);
```

これで、**「本当に新しい情報」**だけが次流れる。

### Step 4: HTTP Request + AIで「要約」という付加価値を

ここが一番のミソ。
単にURLを貼るだけやったら、誰も読まへん。
**「この記事は我々にとって脅威か？」**が一目でわかるようにしたい。

`HTTP Request` ノードを使って、OpenAIのAPIを叩く。
プロンプトはこんな感じ。

> 「以下の記事を読んで、PM視点で3行で要約しろ。特に『新機能』『ターゲットユーザー』に注目せよ。最後に『脅威レベル（高・中・低）』を判定しろ」

これを通すことで、Slackにはこんな感じで流れてくる。

> **🚨 競合A社が新機能リリース**
> [記事リンク]
> - 要約: モバイルアプリにオフライン機能を追加。
> - ターゲット: 現場作業員向けのUX改善。
> - **脅威レベル: 高**（ウチのQ3ロードマップと被ってるで！）

これなら、朝イチでSlack見た瞬間に「うわ、対策会議せな！」って判断できるやろ？

## 導入して変わったこと：PMの仕事は「情報収集」じゃない

このワークフローを稼働させてから、朝の15分が変わった。
情報収集に使ってた時間がゼロになって、その分、**「対策を考える時間」**に充てられるようになったんや。

あと、副次的な効果として、Slackに流れることでエンジニアやデザイナーも「へー、他社はこんなことやってんねや」って興味持ち始めた。
**「情報の民主化」**ってやつやな（カッコよく言いすぎか？）。

## 注意点：APIキーの管理だけは気ぃつけや

n8nは便利やけど、APIキーとかCredentialの設定ミスると痛い目見るで。
ワイも一回、OpenAIのAPIキーの設定ミスって、無限ループでリクエスト送り続けて、朝起きたら**クレジット残高が消滅**してたことある（泣）。
`Split In Batches` ノードを使って、ループ処理には必ずWaitを入れるとか、安全策は講じとこうな。
