**英語が流暢に話せるだけの日本人は、現地では「ただの面白くない奴」で終わる。**

俺を見てみ？
俺の英語力なんて、正直言って **中学生レベル** やで？
ネイティブの早口MTG？半分も聞き取れんわ。
発音？「Th」なんか言えたことない。「S」でゴリ押しや。

それでもな、俺は今、米国のAI企業でプロダクトマネージャー（PM）として働き、現地のエンジニアやデザイナーを指揮して、年収1000万以上を稼いどる。

今日はな、お前らが勘違いしてる「グローバル人材」の幻想をぶち壊して、**「英語力ゼロでも米国企業で無双する、James流・非同期コミュニケーション術」** を全部教えたる。

## 1. 現場のリアル：「流暢な英語」はノイズでしかない

まずな、シリコンバレーとか米国テック企業の現場を美化しすぎや。
みんなが映画みたいに流暢な英語で、ウィットに富んだジョークを飛ばしながら仕事してると思ってるやろ？

**あんなもん、ドラマの中だけや。**

実際の現場はな、もっと殺伐としてるし、もっと「効率」に飢えとるんや。
特にエンジニアはな、**「曖昧な言葉」** が一番嫌いやねん。

「I think, maybe, it would be better if we could make this button a little bit more... pop?」
（えーっと、多分、このボタンをもうちょっとこう... ポップにした方がええんちゃうかな？）

こんなんネイティブ発音で言われてみ？
エンジニアは裏で「F**k」って呟いてるで。**「具体的になんやねん！仕様をよこせ！」** ってな。

ここでな、俺みたいな **「英語雑魚PM」** がどう戦うか。
俺は一切、形容詞を使わん。
その代わり、**「図（Diagram）」** と **「ロジック（Logic）」** を突きつけるんや。

## 2. 武器を持て：「CleanShot X」は英語力の代わりになる

俺が米国企業に入って最初にやったことは、英語の勉強ちゃう。
**スクリーンショットツールの課金** や。

Macユーザーなら黙って **[CleanShot X](https://cleanshot.com/)** を入れろ。
これがないと俺は仕事ができへん。
なんでかって？

**「言葉で説明する」という行為そのものが、コストやからや。**

例えば、UIのバグを報告するとするやろ？

**【英語弱者の戦い方（負けパターン）】**
Slackで一生懸命文章を書く。
"The button on the top right corner overlaps with the header when I scroll down on mobile view..."
（右上のボタンがモバイル表示でスクロールするとヘッダーに重なって...）

これ書くのに何分かかる？5分？10分？
ほんで、伝わるか？
エンジニアは「Which button?」「Wait, let me reproduce it」って返してきて、往復ラリーが始まる。
**これが「英語で負ける」ってことや。**

**【James流・CleanShot X爆撃（勝ちパターン）】**
1. バグってる画面をスクショ撮る。
2. CleanShotのエディタで、バグ箇所に **「矢印」** と **「赤枠」** を引く。
3. 矢印の先に **"Fix this"** って書く。
4. その画像をSlackに貼る。

以上。所要時間、**15秒**。
文章？ **"Mobile view bug. See attached."** だけや。

これでエンジニアはどう思うか？
**「こいつは話が早い。最高や」** ってなるんや。
英語力なんて関係ない。**「視覚情報」の解像度** がすべてや。

俺はな、MTG中もずっとこれやっとる。
誰かが喋ってる間に、議論の内容をリアルタイムで図解して、チャットに貼るんや。
「Are you talking about this flow?（このフローの話してんの？）」って。

すると全員が俺の図を見る。
その瞬間、**会議の支配権は「一番流暢に喋る奴」から「一番わかりやすい図を出した奴（俺）」に移るんや。**

これが **「沈黙の兵器（Silent Weapon）」** や。

## 3. Notionで「非同期の神」になれ

次は **Notion** や。
お前ら、Notionをおしゃれなメモ帳やと思ってないか？
ちゃうで。あれは **「英語を喋らなくて済むための要塞」** や。

米国企業、特にリモートワーク主体のAI企業では、**「同期コミュニケーション（MTG）」は悪** とされる傾向がある。
「MTGを設定する＝相手の時間を奪う」という罪の意識を持て。

俺はな、仕様書（PRD）を書く時、絶対に「質問が出ないレベル」まで書き込む。
英語は箇条書き（Bullet points）でええ。
その代わり、**「なぜやるのか（Why）」「何をやるのか（What）」「どうやるのか（How）」** を徹底的に構造化するんや。

*   **Context (背景)**: 3行で書く。
*   **Problem (課題)**: ユーザーの悲鳴（定性データ）と数字（定量データ）を貼る。
*   **Solution (解決策)**: Figmaの埋め込みと、ロジックツリーを貼る。

これをドカンとSlackに投げる。
"Please review. Async preferred."（レビュー頼む。非同期推奨）と添えてな。

ネイティブPMが「Let's hop on a call（ちょっと通話しようぜ）」って言ってくる間に、俺のドキュメントはエンジニアによって実装が始まっとる。
なぜなら、**エンジニアにとって読みやすいのは、流暢なポエムよりも、構造化された仕様書やからや。**

**「英語が下手だから、ドキュメントで完璧にする」**
このハンディキャップが、結果的に **「ドキュメント駆動開発（Document Driven Development）」** の達人というブランディングに繋がるんや。
皮肉なもんやろ？

## 4. 結論：英語コンプレックスを捨てて、ツールを磨け

ええか、まとめんで。

1.  **英語力で勝負するな。** お前がネイティブに勝てる日は永遠に来ない。
2.  **「視覚情報（Visual）」で殴れ。** CleanShot Xは必須科目や。矢印一本で世界が変わる。
3.  **「非同期（Async）」を極めろ。** 喋るな、書け。Notionで論破しろ。
4.  **「構造化（Structure）」で信頼を勝ち取れ。** 文法ミスなんてどうでもええ。ロジックが通ってれば、相手は動く。

俺たちが目指すのは「英語ペラペラな帰国子女」ちゃう。
**「言葉少なやけど、核心を突く図とドキュメントを出してくる、仕事が爆速な謎のアジア人PM」** や。

こっちの方が、かっこええし、年収も高いで。

英語の勉強に1時間使う暇があったら、**CleanShot Xのショートカットキーを1つ覚えろ。**
Notionのデータベース設計を学べ。
それが、お前を年収1500万の世界へ連れて行く **「最短ルート」** や。

ほな、また。
