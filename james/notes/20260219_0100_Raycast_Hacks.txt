# 【保存版】「キーボードから手を離したら負け」米国Tech PMがRaycastに魂を売った話（設定全公開）

まいど、Jamesです。

今回は**「精神論」禁止**です。「意識高く持とう」とか「朝5時に起きよう」とか、そんなんはどうでもええねん。
我々が欲しいのは**「今すぐ、この瞬間の1秒を削る」**技術やろ？

今日は、僕が愛してやまないMac用ランチャーアプリ**「Raycast」**について、**ガチの実戦設定**を晒します。
正直、これ入れてないMacユーザーは人生の半分損してる……と言いたいとこやけど、半分どころか8割損してます。

## 1. なんでSpotlightじゃあかんのか？

「え、Mac標準のSpotlight（⌘+Space）で十分やん？」
そう思ってた時期が僕にもありました。

でもな、USと日本の時差（13〜16時間）の中で仕事してると、**「今向こう何時やっけ？」**とか**「次のミーティングのZoomリンクどこやっけ？」**っていう**「コンマ数秒の迷い」**が積み重なって、脳のメモリを食いよるんですわ。

Raycastは、ただのランチャーちゃいます。
**「指先だけで完結するコマンドセンター」**です。

## 2. James流・必須Extensions 3選

まずは基本の「Store」から入れられるやつ。これだけで生産性が3倍になる（当社比）。

### ① World Clock（時差ボケ防止の命綱）
US西海岸、東海岸、日本、ロンドン……チームが世界中に散らばってるPMにとって、時差計算は死活問題。
いちいちGoogleで「pst now」とか打ってる場合ちゃいます。

- **設定:** Raycastの設定画面で `w` をエイリアスに登録。
- **使い方:** `⌘ + Space` → `w` → `Enter`
- **結果:** 一瞬で登録した全都市の時間がリスト表示される。

これの何がエグいって、**「会議の候補日時をコピーする機能」**があること。
「向こうの火曜15時」をサッとクリップボードに入れてSlackに貼り付ける。これだけで寿命が延びます。

### ② Clipboard History（記憶の外部化）
これがないと生きていけへん。
過去にコピーしたテキスト、画像、リンクを全部覚えててくれるやつ。

- **James流設定:** ホットキーを `⌥(Option) + V` に設定。
- **活用シーン:**
  - Slackでさっきの指示内容をもう一回貼りたい時
  - 2段階認証コードをコピペする時
  - スクショを撮って、その場では貼らずに後でまとめて貼る時

「あれ、さっきコピーしたURLどこいった？」っていう絶望がゼロになります。ゼロです。

### ③ DeepL Translate（英語弱者の守護神）
僕もPMやってますけど、ネイティブの超早口スラング英語はキツい時があります。
そんな時はこれ。

- **設定:** 翻訳したいテキストを選択して、`⌘ + Space` → `dl`（エイリアス設定推奨）。
- **挙動:** 選択範囲が勝手に翻訳されて、翻訳結果が**「ポップアップ」**で出るか、**「クリップボードに上書き」**される。

いちいちDeepLのアプリ立ち上げて、コピペして……っていう動作が不要。
**「選択 → ホットキー」**だけで翻訳完了。これでSlackのレスポンス速度が爆上がりします。

## 3. 【上級編】自作Script Commandsで「定型業務」を殺す

ここからが本番です。Raycastの真骨頂は**「自分でスクリプトを書ける」**こと。
僕が実際に使ってる、泥臭いけど最強のスクリプトを2つ紹介します。

### ① 「Zoom会議、強制Join」
カレンダーの通知が来ても、作業に集中してて「あ！忘れてた！」ってなること、ありません？
僕はあります。なので、**「次の会議のURLを自動で取得して開く」**コマンドを作りました。

Bashで書けます。これを `Next Meeting.sh` とかで保存してRaycastに登録。

```bash
#!/bin/bash

# @raycast.schemaVersion 1
# @raycast.title Join Next Meeting
# @raycast.mode silent

# 実際はcalのAPI叩いたり、AppleScriptでカレンダーから取得する処理が入りますが、
# 簡易的には「直近のZoomリンク」をテキストファイルから引っこ抜くとかでもOK。
open "zoommtg://zoom.us/join?confno=123456789" 
# ↑ここを動的に取得するように組むのがコツ
```

僕はこれを `nm` (Next Meeting) で登録してます。
会議開始1分前に `⌘+Space` → `nm`。これだけ。カレンダーアプリを探す必要なし。

### ② 「GitHubのPR、一括承認（嘘です、確認用）」
PR（Pull Request）のレビュー依頼が溜まると死にたくなりますよね。
ブラウザでタブをポチポチ開くのが面倒なので、**「自分宛のレビュー待ちPRをリスト化して、Enterで開く」**スクリプトを使ってます。

これは `gh` コマンド（GitHub CLI）と組み合わせます。

```bash
#!/bin/bash

# @raycast.title Review Requests
# @raycast.mode fullOutput

gh pr list --search "review-requested:@me" --json number,title,url --template '{{range .}}{{printf "#%v %v\n%v\n\n" .number .title .url}}{{end}}'
```

これを叩けば、Raycastのウィンドウ内に自分宛のPRがズラッと並ぶ。
「うわ、5件も溜まってるやん……」って絶望するのも、ブラウザを開かずに済みます（何の解決にもなってへんけど、確認は速い）。

## 4. 失敗談：やりすぎて「設定沼」にハマるな

実は僕、最初は**「ありとあらゆるショートカット」**を登録しすぎて自爆しました。
「Jiraを開く＝ `ji`」「Confluenceを開く＝ `cf`」「Gmail作成＝ `gm`」……。

**覚えられへんねん。**

結局、脳のメモリを節約するためにツール入れてるのに、ショートカット覚えるのにメモリ使ったら本末転倒ですわ。

**【解決策】**
- **「1日10回以上やる動作」**だけを登録する。
- 覚えられないものは、無理にショートカット化せず、普通に検索して打つ。Raycastの学習機能が優秀なんで、数回打てば勝手に上位に来ます。
