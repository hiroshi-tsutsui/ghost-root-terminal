# 【実話】「英語力」よりも「図解力」。TOEIC 500点でも米国AI企業でPMが務まる理由（と、そのための武器）

まいど、Jamesです。

いきなりですが、結論から言います。

**「英語ができないから、外資系テック企業は無理」**

これ、真っ赤な嘘です。いや、嘘というか「言い訳」にしかなってへん。

僕のTOEICスコア、知ってますか？
ええかっこしたいから言いたくないけど、渡米前は **500点台** でした。
中学生レベルの文法と、受験英語の残骸みたいな単語力。

それでも今、僕はシリコンバレーのAI企業でPM（プロダクトマネージャー）として、年収1000万以上をもらいながら働いています。エンジニアやデザイナー、データサイエンティストといった「天才たち」を動かしています。

**「なんでそんなことが可能なん？」**

それは、僕が **「流暢な英語（Fluent English）」** を捨てて、 **「視覚的な暴力（Visual Violence）」** という武器を手に入れたからです。

今日は、日本人が陥りがちな「英語学習の罠」をぶち壊し、明日から使える「非同期コミュニケーション」の極意を叩き込みます。これは、PMだけやなく、全てのエンジニア、デザイナーにも通じる生存戦略です。

---

## 1. 「英語の会議」は、ただのノイズや

日本人の真面目な君は、こう思ってるはずです。

> 「Zoom会議で、ネイティブの早い英語を聞き取って、即座にかっこいい英語で言い返さなアカン」

これ、無理ゲーです。諦めてください。
ネイティブ同士の議論なんて、F1レースみたいなスピードです。僕らがそこに三輪車（カタコト英語）で突っ込んでも、クラッシュするだけ。

僕も最初は必死でした。
会議の録画を何度も見直して、議事録を書いて、単語帳を作って...。
でも、ある日気づいたんです。

**「あいつら、会議で何一つ決まってへんやん」**

そう、アメリカ人も会議が下手なんです。
喋るのが好きなだけ。
「I think...」「Awesome!」「Make sense」とか言い合ってるけど、結局 **「誰が、いつまでに、何をやるか」** が曖昧なまま終わることが多い。

ここで、僕ら「非ネイティブ」の勝機が生まれます。

## 2. 英語力を凌駕する「CleanShot X」という暴力

僕が会議中に何をしてるか。
必死にリスニングしてるフリをして、**画面のスクショを撮りまくってます。**

そして、会議が終わった瞬間に、あるツールを使って「図解」を作り、Slackに投げるんです。

そのツールが、**CleanShot X** です。

これ、ただのスクショツールやない。「意思伝達の加速装置」です。

### 実際に僕がやってるフロー

1.  **会議中、議論が紛糾した瞬間の画面をスクショ（`Cmd + Shift + 4` じゃない、CleanShotのショートカットや）。**
2.  **その場で「矢印（Arrow）」と「テキスト（Text）」を書き込む。**
    *   「Here is the problem（ここがバグ）」
    *   「Fix this logic（このロジック直して）」
    *   「Change to B pattern（B案に変えて）」
3.  **GIF動画も撮る。**
    *   再現手順を口で説明するのは無理でも、動画なら一発。
    *   CleanShotなら、クリックした場所が波紋で光るし、キー入力も画面に出る。
4.  **リンクを発行してSlackに貼る。**

そして、こう書き添えます。

> **"Just to confirm, is this what we agreed on?" (確認やけど、合意したんこれであってる？)**

これだけ。
文法なんてどうでもいい。
**「画像」という事実は、誰にも否定できません。**

ネイティブの流暢な議論よりも、僕の貼った「矢印付きの画像1枚」の方が、圧倒的に情報量が多い。
エンジニアも「Oh, exactly! James gets it.」ってなる。

これが **「視覚的な暴力」** です。
英語力の欠如を、解像度の高い「図」で殴って埋めるんです。

## 3. Notionは「英語の教科書」やない、「仕様の聖書」や

もう一つの武器が **Notion** です。
これも使い方が間違ってる人が多い。

「綺麗なドキュメント」を書こうとしてませんか？
あんなもん、誰も読みません。

僕が意識してるのは **「構造化（Structure）」** だけです。

*   **Bullet Points（箇条書き）** を多用する。
*   **Toggle（トグル）** で詳細を隠す。
*   **Mermaid記法** でフローチャートを書く。

例えば、仕様を説明するとき。

**【ダメな例】**
> Since the user data is heavy, we should create a cache layer to improve performance... (長い！読む気失せる！)

**【James流】**
> **Performance Issue & Solution**
> *   **Problem**: User data load takes > 3s.
> *   **Solution**: Implement Redis Cache.
>     *   **TTL**: 600s
>     *   **Key**: `user_{id}_profile`

これなら、中学生英語で書けます。
そして、ネイティブもこっちの方が読みやすいんです。

**「英語を書く」んじゃなくて、「ロジックを置く」感覚。**
プログラミングコードを書くのに近いです。
これなら、僕ら日本人エンジニア・PMの方が得意な領域です。

## 4. 1秒を削るための投資をケチるな

CleanShot X は有料（$29/one-time くらい）です。
「え、OS標準のスクショでよくない？」って思った人。

**その思考が、年収アップを阻んでます。**

*   スクショを撮る。
*   プレビューが開くのを待つ。
*   注釈ツールを選ぶ。
*   保存する。
*   Slackにドラッグ＆ドロップする。

この一連の動作に「1分」かかってるとしたら、CleanShot X なら「10秒」で終わります。
1日10回やれば、500秒の差。
1ヶ月で...1年で...。

何より、**「面倒くさいから図解しない」という心理的ハードルが一番の敵** です。
ツールが快適なら、息をするように図解できる。
それが「英語ができないのに、なぜかコミュニケーションが円滑なPM」という評価に繋がります。

## 5. 結論：言葉の壁は「解像度」で越えろ

英語の勉強をするな、とは言いません。
でも、TOEICの点数を100点上げるのに半年かけるくらいなら、**「図解スキル」と「ツールへの投資」を今すぐやってください。**

明日から、SlackやTeamsでの会話を全て「スクショ」と「箇条書き」に変えてみてください。

「Jamesの言う通りやったわ。インド人のエンジニアと爆速で仕様が固まったわ」

そう思ったら、またこのnoteに戻ってきてください。
その時、君はもう「日本のイチ担当者」やなく、「グローバルで戦えるPM」への第一歩を踏み出しています。

ほな、また。
