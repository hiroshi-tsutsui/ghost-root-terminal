# 【実話】「英語力」よりも「図解力」で殴れ。TOEIC 900点超えのエリートが現場で沈む理由。

## 0. はじめに：英語の勉強、今すぐやめなはれ。

「Jamesさん、米国企業で働くために、まずは英会話スクールに通おうと思ってます」

これ聞くたびに、僕、ほんまに頭抱えるんよ。
いや、気持ちはわかる。わかるで。
「英語圏で働く＝流暢な英語が必要」っていう**呪い**にかかっとるんやろ？

はっきり言うとくわ。
**その「完璧な英語」を目指す努力、ROI（投資対効果）が最悪や。**

僕が現場で見てきた「消えていく日本人」の共通点。
それは**「正しい英語を話そうとして、地蔵になる」**こと。
会議で発言せず、後から綺麗な議事録を出してくるタイプ。これ、米国テック企業のスピード感では「存在しない」のと同じ扱いやねん。

逆に、TOEICなんて受けたこともない、文法めちゃくちゃなインド人エンジニアが、拙い英語とホワイトボード一枚でチームを支配してる光景、死ぬほど見てきた。

今日は、僕が米国AI企業のPMとして生存するために編み出した**「英語力ゼロでも議論を制する」泥臭いハック**を全部公開する。
これは「英語の勉強法」やない。「コミュニケーションの格闘術」や。

## 1. 「綺麗な英語」は、ただの自己満足。

まず、大前提を破壊させてくれ。
君らが学校で習った「Subject + Verb + Object...」みたいな構文、現場では**誰も気にしてへん**。

僕の同僚のエンジニア（年収3000万クラス）のSlack、こんなんやで。

> *"API broken. Fix now? or later?"*
> *"No logic here. Why?"*

これ、中学1年生レベルやん？
でも、これで仕事は回るし、彼はチームのキーマンや。

なぜか？
**「意図」が1秒で伝わるから**や。

日本人が陥りガチなのが、これ。

> *"I think it would be better if we could consider fixing the API because it seems to be broken..."*

丁寧やけど、**ノイズが多い**。
PMとして一番重要なのは「品格」やない。「決定（Decision）」のスピードや。

「流暢に話す」ことと「仕事を進める」ことは、全く別のスキルやと認識してほしい。
僕らが目指すべきは、**「ブロークンでも、相手を動かせる英語」**や。

## 2. 武器は「口」やない。「画面」や。

じゃあ、文法も発音も自信がない僕らが、どうやってネイティブと対等に渡り合うか？
答えはシンプル。**「視覚情報（Visual）」に全振りする**ことや。

人間の脳は、音声処理よりも画像処理の方が圧倒的に速い。
特に、英語が第二言語の人間にとって、「耳」だけで勝負するのは、目隠ししてボクシングするようなもんや。自殺行為やで。

僕が会議で必ずやってる「鉄の掟」がある。

**「画面共有（Screen Share）の主導権を絶対に渡さない」**

会議が始まった瞬間、誰よりも早く「Can I share my screen?」って言って、画面を奪う。
これだけで、勝率は8割上がる。

なぜか？
画面共有してる人間が、その場の**「コンテキスト（文脈）」を支配できる**からや。

### CleanShot X を「神速」で使いこなせ

ここで登場するのが、僕が口を酸っぱくして推してる**『CleanShot X』**や。
（これ使ってないMacユーザーのPMは、正直モグリやと思ってる）

会議中、相手が複雑な要件を早口でまくし立ててくるとする。
「Okay, hold on」って言って、その場でホワイトボードツール（MiroとかFigJam）を開く。
あるいは、Notionのページを開く。

そして、相手が言ったことを**その場で図にする**んや。

- 四角を描く（Box）
- 矢印を引く（Arrow）
- テキストを叩き込む（Text）

*"You mean, this data flows from A to B, right?"*（このデータはAからBに流れるってこと？）

画面上の図を指差しながら、こう聞く。
これなら、**「This」と「That」だけで会話が成立する**。
高度な文法なんていらん。

相手も「No, no, from A to C directly!」って、図を修正してくれる。
この**「共通の絵」**を作れた時点で、君の勝ちや。
君は「英語ができない日本人」から、**「議論を整理してくれたファシリテーター」**に昇格する。

## 3. 非同期（Async）で殴れ。

もう一つの武器が**「非同期コミュニケーション」**や。
リアルタイムの口頭バトル（会議）は、ネイティブの独壇場。
そこで勝とうとするのが間違いや。

僕らの戦場は**「ドキュメント」**にある。

会議の前、あるいは後に、圧倒的な解像度のドキュメントを投下する。
Amazonとかでも徹底されてるけど、**「読む」文化**を自分の周りに作るんや。

### James流・ドキュメント構成の「型」

僕がSlackやNotionで提案するときは、絶対に長文を書かない。
**「箇条書き」と「太字」と「画像」**だけで構成する。

悪い例：
> I analyzed the user data and found that many users are dropping off at the signup page. So I think we should improve the UI...

良い例：
> **Proposal: Fix Signup Drop-off**
>
> **Problem:**
> - Drop-off rate: **65%** (High!)
> - Main friction: "Password confirmation" field.
>
> **Solution:**
> - Remove "Password confirmation".
> - [Insert Screenshot of new UI mockup]
>
> **Impact:**
> - Est. +15% conversion.

これなら、英語力なんて関係ない。
**「数字（Logic）」と「画像（Visual）」**は世界共通言語やからな。

ネイティブが「Uh... Well...」って喋ってる間に、君はこのドキュメントをSlackに貼り付ける。
*"Please check this. I summarized the plan."*

これで、議論のベースは君のドキュメントになる。
**「書く」スピードと質で、「話す」ハンデを覆す**んや。

## 4. 結論：PMの価値は「翻訳」にある。

勘違いせんといてほしいのは、僕らがやってるのは「日本語から英語への翻訳」やない。
**「曖昧なアイデア」から「実装可能な仕様」への翻訳**や。

エンジニアが求めてるのは、流暢な英語で語られる夢物語やない。
「で、何を作ればいいの？」という明確な指示（Spec）や。

それを伝えるのに、シェイクスピアみたいな英語はいらん。
**四角と矢印、そして箇条書き。**
これだけで、年収1000万の仕事はできる。

英語コンプレックスを捨てろ。
その代わりに、**作図ツールとスクショツール（CleanShot X）を極めろ**。
キーボードショートカットを指に覚え込ませろ。

会議で誰よりも速く図解し、誰よりも速くドキュメント化する。
その「速度」こそが、君の英語力をカバーし、チームからの信頼を勝ち取る最強の武器になる。

ほな、今すぐCleanShot Xの設定見直して、次の会議で画面奪い取ってきや。
