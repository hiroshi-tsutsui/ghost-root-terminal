特にUSのテック企業で働いてると、セキュリティ要件がアホほど厳しい。
「API Keyはローカルに保存するな」「2FAは毎回求めろ」「90日でパスワード変えろ」...

そこで使うのが **『1Password CLI (`op`)』** や。

これ、ただの「コマンドでパスワード取れるやつ」やと思ってるなら大間違いやで。
これは**「Terminalそのものを生体認証デバイス化する」**ための武器や。

## 🚫 失敗談：`.env` ファイルの墓場

昔のワイはこうやった。

1. プロジェクトごとに `.env` ファイルを作る。
2. `AWS_ACCESS_KEY_ID=AKIA...` とか書く。
3. 間違ってGitHubにPushする。
4. **Security TeamのBotからSlackで「Fxxk」ってDMが飛んでくる。**
5. キーをローテーションさせられて半日潰れる。

...あるあるやろ？（ないとは言わせんぞ）

ほんで、「暗号化しよう」とか「`git-crypt`使おう」とかやるんやけど、結局復号する手間がダルい。
面倒くさいセキュリティは、誰も守らんのよ。

## ✅ 解決策：`op run` で "透明な" 環境変数を渡す

1Password CLI の真髄は `op run` にある。

これを使うと、**「コマンド実行時の一瞬だけ、メモリ上に環境変数を展開して、終わったら消える」** という神仕様が手に入る。
ファイルには一切保存せん。だから漏れようがない。

### Step 1: 1Password CLI のインストールと連携

まず `brew` で入れる。GUIアプリも入れて、**「Integrate with 1Password CLI」** にチェック入れるのを忘れんようにな。これやらんとTouch ID使えへんから意味ないで。

```bash
brew install 1password-cli
```

### Step 2: Secret Reference を取得する

1Passwordのアプリで、使いたいAPI Keyの項目を右クリックして「Copy Secret Reference」を選ぶ。
中身はこんな感じの呪文や。

`op://Private/AWS_Production/access_key_id`

### Step 3: `.env.template` を作る（本物は作らない）

ここがミソや。`.env` ファイルに直接キーを書く代わりに、この「呪文」を書く。

```bash
# .env.local
AWS_ACCESS_KEY_ID="op://Private/AWS_Production/access_key_id"
AWS_SECRET_ACCESS_KEY="op://Private/AWS_Production/secret_access_key"
```

これならGitにコミットしても（まあアカンけど）最悪、中身はただのパスやから実害は少ない。

### Step 4: `op run` で召喚する

ほんで、コマンドを実行するときにこう打つんや。

```bash
op run --env-file="./.env.local" -- aws s3 ls
```

実行した瞬間、Touch IDのプロンプトが出る。
**「指を置く」→「認証OK」→「環境変数が注入される」→「AWSコマンド実行」**

この間、0.5秒。
クリップボードも汚さない。ターミナルの履歴にも残らない。
**完全に「透明」な認証や。**

## 🚀 James流：Aliasで "魔法" にする

いちいち `op run -- ...` とか打ってられへんよな？
ワイは `.zshrc` にこう書いてる。

```bash
# Prod環境に繋ぐときは "prod" って打つだけ
alias prod='op run --env-file=~/.config/op/prod.env --'

# 使用例
prod tensorflow_training_job.py
```

これで、「指を置くだけで本番環境のDBに接続してクエリ投げる」ことができる。
横で見てる同僚からしたら**「え、お前今パスワード打ってなくない？ ハッカー？」**ってなること請け合いや。

## 💡 おまけ：MFA（ワンタイムパスワード）も自動入力

CLIでログインするとき、MFA求められることあるやろ？
あれもスマホ取り出して、Google Authenticator開いて...ってやってる場合ちゃう。

```bash
# 2FAコードをクリップボードにコピーして貼り付け待ち状態にする
alias otp='op item get "AWS Production" --otp | pbcopy'
```

これも `op` コマンド一発や。Raycastと組み合わせてもええけど、Terminalから手を離したくない時はこっちのが速い。

## まとめ

1. **`.env` に秘密情報を書くな。** `op://` の参照を書け。
2. **`op run` を使え。** メモリ上だけで展開しろ。
3. **Touch IDを信じろ。** 指紋こそが最強のパスワードや。

これで「セキュリティ事故」と「コピペの手間」の両方から解放される。
