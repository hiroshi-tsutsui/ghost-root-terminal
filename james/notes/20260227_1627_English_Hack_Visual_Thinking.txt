# 【実録】英語力ゼロから米国テックPMになれたのは「図解力（Visual Thinking）」のおかげやった話。

まいど、Jamesです。

今回は「英語」の話をします。
これ読んでるエンジニアやPM志望のみなさん、正直に言うてください。

**「英語がペラペラにならんと、外資や海外就職は無理」**

って思ってませんか？

それ、完全に**間違い**です。断言します。
もし僕がネイティブ並みの英語力を目指してから転職活動してたら、今頃まだ大阪の片隅で単語帳めくってたでしょうね。一生そこから抜け出せんかったと思います。

僕のTOEICスコア？ 聞かんといてください（笑）。
いや、今はそこそこありますけど、渡米当初はひどいもんでした。「Function」の発音が通じず、会議で3回聞き返されて「もうええわ、Slackで送るわ」ってなった時の絶望感、味わったことありますか？

それでも今、僕は米国テック企業でPMとして、ネイティブのエンジニアやデザイナーを動かしてプロジェクト回してます。年収も日本の時の倍以上です。

なんでそんなことが可能なんか。
答えはシンプル。

**「喋る」んやなくて「見せる」からや。**

今日は、僕が英語のハンデをひっくり返すために編み出した**「Visual Communication（視覚的暴力）」**という生存戦略について、泥臭い実例付きで解説します。これさえあれば、TOEIC 600点だろうが何だろうが、明日からグローバルチームで戦えます。

---

## 1. 言葉は「誤解の源泉」やと諦める

まずマインドセットを変えてください。
我々ノンネイティブが、ネイティブ相手に「言葉だけ」で勝負しようとするのがそもそも間違いなんです。

日本語でもそうでしょう？
要件定義の会議で、口頭だけで「ここのボタンを押したら、ええ感じでポップアップが出て…」とか言うPM、信用できますか？ できませんよね。絶対あとで「言った言わない」の揉め事になります。

ましてや英語です。
前置詞ひとつ、冠詞ひとつでニュアンスが変わる言語で、完璧な説明をしようとするのは、**「竹槍でB-29を落とす」**くらい無謀な戦いです。

僕が米国に来て最初にやったミス。
それは、必死にスクリプトを書いて、流暢に喋ろうと練習したことでした。

会議当日、僕は準備した完璧なイントロダクションを披露しました。
"Today, I would like to propose a new feature regarding the user onboarding process..."

みんな静かに聞いてくれました。
でも、その後のQ&Aで地獄を見ました。

エンジニアA: "So, does this impact the backend schema for the `user_profile` table?"
エンジニアB: "Wait, are we talking about the mobile view or the desktop dashboard?"

質問の嵐。しかも早口で聞き取れない。
僕はパニックになり、"Ah... yes... maybe..." と曖昧な返事を繰り返して、その場の信頼を一瞬で失いました。

その夜、悔しくて枕を濡らしながら（嘘です、酒飲んでふて寝しました）、僕は悟りました。
**「俺の英語、誰も聞いてへんわ」**と。

彼らが知りたいのは、「俺の流暢な英語」やない。「何を作るのか」「どう動くのか」という**「仕様（Spec）」**だけなんや。

そこから僕は、**「喋るPM」**を辞めました。
**「見せるPM」**にジョブチェンジしたんです。

## 2. 英語力を「画力」で殴る

僕が徹底したのはこれです。

**「1分喋るくらいなら、1枚の図を描く」**

英語が下手な人間が、会議で主導権を握る唯一の方法。
それは、**全員の視線を「自分」ではなく「画面」に集中させること**です。

会議が始まった瞬間、僕は何も言わずに画面共有をします。
そこに映し出すのは、テキストの箇条書きではありません。

- **Figmaのプロトタイプ（動くやつ）**
- **Mermaidで書いたシーケンス図**
- **Miroに貼ったユーザーフロー**

そして、こう言います。
**"Let's look at this diagram."**

これだけです。
これだけで、場の空気が変わります。

全員が図を見ます。
エンジニアは「あ、ここのAPIコール、非同期じゃないとマズいな」とコードをイメージし始めます。
デザイナーは「このモーダルの遷移、ちょっとUX悪いかも」と体験を考え始めます。

僕の英語？
**"Here." "Click this." "Data flows to here."**
これくらいで十分です。中1レベルです。

でも、議論はめちゃくちゃ進みます。なぜなら、**「共通の事実（図）」**が目の前にあるから。
誤解の入る余地がないんです。

これが僕の言う**「Visual Communication」**です。
英語という不完全なプロトコルを捨てて、**「図解」という世界共通プロトコル**で戦うんです。

---

## 3. James流「視覚的暴力」の三種の神器

じゃあ具体的に何を使えばええんか。
僕が毎日使い倒している、英語弱者救済のためのツールを3つ紹介します。
これを使いこなせたら、明日からあなたの英語力は実質ネイティブ並みです。

### ① CleanShot X（Mac必須）
これ持ってないPMは、武器持たずに戦場行くようなもんです。

ただのスクショツールやと思ったら大間違いです。
これの真骨頂は**「注釈（Annotation）」**の速さです。

バグ報告をする時、テキストで
"The submit button is not aligned properly on the settings page."
とか書いてませんか？
そんなん書いてる間に、CleanShot Xなら：

1. `Cmd + Shift + A` でスクショ撮る。
2. 矢印を引く。
3. "Fix this" と書く。
4. 画像リンクをSlackに貼る。

これ5秒で終わります。
エンジニアからしても、長文の英語を読むより、矢印のついた画像のほうが100倍ありがたいんです。
**「英語を書く時間」を「画像を撮る時間」に置換する。** これが最強の時短術です。

### ② Mermaid.js（Notion / GitHub）
エンジニアと話す時は、フローチャートやシーケンス図が最強です。
でも、パワポやCacooでいちいち図を描くのはダルいですよね？

そこで **Mermaid** です。
テキストベースで図が書けるやつです。

NotionでもGitHubのIssueでも、
```mermaid
sequenceDiagram
    User->>Frontend: Click Button
    Frontend->>Backend: POST /api/v1/submit
    Backend-->>Frontend: 200 OK
```
って書くだけで、勝手に綺麗な図になります。

これをMTGの前に用意しておくだけで、エンジニアからの信頼度が爆上がりします。
「こいつ、英語はカタコトやけど、ロジックは完璧やな」と思わせたら勝ちです。
英語の文法ミスなんて、誰も気にしなくなります。

### ③ Loom（非同期ビデオ）
これが今のところ最強のソリューションかもしれません。
時差のあるチームや、複雑な仕様説明には **Loom** 一択です。

画面を操作しながら、喋って録画するツールです。
「え、喋るんやったら英語力いるやん」と思いました？

逆です。
**「撮り直しができる」**んです。

リアルタイムの会議だと、噛んだり詰まったりしたらパニックになりますが、Loomなら何回でもリテイクOKです。
しかも、自分の顔を小さくワイプで出しながら、画面のここを見てくれとマウスで指し示せる。

僕は重要な仕様変更の時は、必ず3分以内のLoom動画を撮って、Slackに投げます。
**"Please watch this video for the details."**

これなら、相手も倍速で見れるし、聞き取れなかったら巻き戻せる。
お互いにとってWin-Winなんです。
しかも「Jamesはいつも分かりやすい資料を用意してくれる」というブランディングにもなります。

## 4. 結論：英語コンプレックスを「武器」に変えろ

英語ができないことを嘆くのは、もう終わりにしましょう。
むしろ、**「英語ができないからこそ、他の手段を研ぎ澄ますチャンス」**と捉えてください。

ネイティブのPMは、言葉でなんとかしようとして、逆にドキュメントが雑だったりします。
そこが我々の勝機です。

**「英語は下手やけど、資料は世界一わかりやすいPM」**
**「会議では無口やけど、出す図解が鋭すぎて議論を支配するPM」**

これを目指してください。
そのポジションを確立した時、あなたの年収は日本円の枠を飛び越えます。

英語の勉強をするなとは言いません。
でも、単語帳を1ページ進める時間があるなら、**CleanShot Xのショートカットを1つ覚えるか、Mermaidの構文を1つマスターしてください。**

そのほうが、間違いなく「現場で使える戦闘力」になります。

僕が米国で生き残れたのは、TOEICのおかげでも、英会話スクールのおかげでもありません。
**「伝われば何でもええ」という開き直り**と、**「視覚情報で相手を殴る」という執念**のおかげです。

ほな、Figma開いて図でも描きましょうか。
現場からは以上です。

James
